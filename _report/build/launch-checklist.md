# Чек-лист запуска — портал «Зодиакум»

> Составлен по результатам фазы Ф8 (см. [f8-отчёт.md](f8-отчёт.md)). Разбит на пункты, которые
> **заказчик выполняет вручную** (инфраструктура, юридические факты, живые ключи) — агент их не
> фабрикует (§8 конвенций реализации). Каждый пункт — с указанием, что именно нужно сделать и
> где в коде это подключается.

## 1. Юридический гейт (блокирует публичный запуск)

      всех 4 документах (согласие на обработку ПД, согласие на маркетинг, политика ПД, оферта).
- [ ] **Уведомление в Роскомнадзор** — подать (если ещё не подано) и вписать факт/дату вместо
      плейсхолдера в `PRIVACY_POLICY_TEXT_RU`/`PD_PROCESSING_CONSENT_TEXT_RU`.
- [ ] **Юридическая вычитка** всех 5 текстов (`PD_PROCESSING_CONSENT_TEXT_RU`,
      `MARKETING_CONSENT_TEXT_RU`, `PRIVACY_POLICY_TEXT_RU`, `OFFER_TEXT_RU`,
      `DISCLAIMER_TEXT_RU`) — тексты помечены как «черновик, не имеет юридической силы»
      (`apps/web/lib/LegalDocPage.tsx`), пометка снимается юристом.
- [ ] **Условия возврата** (`OFFER_TEXT_RU` §5) — заполнить конкретные правила возврата средств
      за разовые услуги/подписку.
- [ ] **Проверка товарных знаков** «Матрица судьбы» / «метод Ладини» в Роспатенте (флагировано ещё
      в Ф6, `_report/build/f6-отчёт.md`, doc-комментарий `packages/shared/src/schemas/order.ts`
      про PDF-продукт «Матрица судьбы»). При конфликте — нейтральный нейминг PDF-продукта и
      публичных страниц (`/matrica-sudby`, `PDF_PRODUCT_CATALOG.matrix_full_pdf`).
- [ ] **Мониторинг законопроекта** о запрете рекламы эзотерических услуг (редакция от 13.02.2026,
      см. `docs/research/05-рынок-услуги-юридика.md`) — до принятия решения не запускать платную
      рекламу астрологических услуг без юридической консультации.
- [ ] **Cookie-баннер** — не реализован в Ф0-Ф8 (портал использует только необходимые
      httpOnly-cookies для сессии/CSRF, см. `apps/api/src/auth/cookies.ts`) — юрист подтверждает,
      достаточно ли информационного баннера о необходимых cookies или нужен полноценный
      cookie-consent виджет (зависит от того, подключается ли аналитика с трекинговыми cookies —
      см. п. 4).

## 2. Платёжный контур (ЮKassa)

- [ ] Зарегистрировать мерчанта в ЮKassa, пройти андеррайтинг (см. риск в
      `docs/research/05-рынок-услуги-юридика.md`: «одобрение зависит от андеррайтинга конкретного
      мерчанта»; CloudPayments/Т-Банк содержит эзотерику в списке запрещённых категорий —
      **первичный провайдер — ЮKassa**, реализовано именно под неё).
- [ ] Получить `YOOKASSA_SHOP_ID`/`YOOKASSA_SECRET_KEY`, задать `PAYMENTS=yookassa` в проде.
- [ ] Активировать СБП в личном кабинете ЮKassa (см. doc-комментарий
      `packages/shared/src/ports/yookassa-payment-provider.ts` — ЮKassa сама предлагает способ
      оплаты на экране подтверждения, отдельного кода не требуется, только настройка мерчанта).
- [ ] **Настроить URL вебхука** в личном кабинете ЮKassa: `{APP_URL}/api/v1/webhooks/yookassa`.
- [ ] **IP allowlist** вебхуков на уровне инфраструктуры (Caddy/файрвол) — реальная защита от
      подделки вебхука на проде (наш HMAC-слой `YOOKASSA_WEBHOOK_SECRET` — дополнительный, см.
      doc-комментарий `packages/shared/src/schemas/billing.ts`
      `WEBHOOK_SIGNATURE_HEADER`), опционально задать `YOOKASSA_WEBHOOK_SECRET`.
- [ ] **Настройка триала без немедленного списания** — MVP-упрощение (см. doc-комментарий
      `apps/api/src/routes/subscriptions.ts`): первый платёж подписки сейчас ВСЕГДА проводится
      через `createPayment`, даже для тарифов с триалом. Для реальной схемы «7 дней без
      списания» нужно либо настроить у ЮKassa привязку карты без списания (верификационный
      холд), либо перенести первый платёж на конец триала — согласовать с ЮKassa при
      онбординге и, при необходимости, доработать `subscriptions.ts`.
- [ ] **54-ФЗ / онлайн-касса**: подключить встроенную фискализацию ЮKassa (см.
      `CreatePaymentInput.receipt` в `packages/shared/src/ports/payment-provider.ts`,
      `receiptStatusEnum` в `packages/db/src/schema/enums.ts`) — либо кассу самозанятого/ИП
      («Мой налог»), согласовать реальную ставку НДС/номенклатуру с бухгалтерией (в коде
      дефолт `vat_code=1`, «без НДС» — placeholder, не аудированное решение).
- [ ] **E2E-прогон в песочнице ЮKassa** (verification-пункт промта Ф8): разовый платёж → чек →
      PDF доступен; подписка → триал → списание → отмена → доступ закрыт по концу периода;
      возврат → доступ отозван. Не выполнено в этой фазе (нет живых ключей) — юнит-покрытие
      логики идемпотентности/статусов есть (`apps/api/src/billing/webhook-processor.test.ts`,
      `apps/api/src/routes/billing.test.ts`).

## 3. Инфраструктура

- [ ] **Прод-БД** — Yandex Managed PostgreSQL (152-ФЗ: ПД россиян — в РФ с первого публичного
      запуска, см. `docs/architecture/21-техническая-архитектура.md` §7). Применить миграции с
      нуля: `pnpm db:migrate && pnpm db:seed` (все 11 сидов, включая новый
      `drizzle/seed/0011_plans_experiments.sql`).
- [ ] **S3** — Yandex Object Storage, задать `STORAGE=s3` + `S3_*` переменные (сейчас
      `MemoryObjectStorage`, PDF-файлы теряются при рестарте — годится только для дев/демо).
- [ ] **Секреты прод-контура** — сгенерировать боевые `JWT_PRIVATE_KEY`/`JWT_PUBLIC_KEY`
      (EdDSA), `PD_ENCRYPTION_KEY` (32 байта base64), `COOKIE_SECRET`; `parseConfig` падает на
      старте в `NODE_ENV=production`, если оставлены dev-дефолты (см. `packages/shared/src/
      config.ts`) — это встроенная защита, а не опция.
- [ ] **Бэкапы БД** — управляемые снапшоты Yandex MPG + логическая репликация, RPO ≤ 24ч;
      **отрепетировать восстановление** из бэкапа (цель — ≤30 мин, см. верификацию промта Ф8),
      зафиксировать шаги и время в отдельном runbook-документе перед запуском.
- [ ] **Мониторинг/алерты** (`docs/architecture/21-техническая-архитектура.md` §9): prom-client
      `/metrics` (уже отдаётся `apps/api`), Sentry/GlitchTip для ошибок, алерты на долю 5xx,
      отставание очереди pg-boss, провал ежедневной генерации гороскопов, рост стоимости LLM —
      подключить внешний коллектор (Prometheus/Grafana) и алертинг, в коде это не настроено
      (только метрики отдаются).
- [ ] **Нагрузочный тест** (autocannon: 200 RPS программатика из кэша, 20 RPS API, цель p99 SSR
      <300 мс) — не выполнен в этой фазе (требует поднятого прод-подобного стенда).
- [ ] **LLM-провайдер** — задать `LLM_PROVIDER`/`ANTHROPIC_API_KEY` (или другой) для реальных
      текстов интерпретаций/квиза/вступлений к индивидуальным прогнозам (сейчас `stub` — честные
      детерминированные тексты с пометкой, портал не пустой, но черновой).
- [ ] **SMTP** — задать `MAILER=smtp` + `SMTP_*` для реальной доставки писем жизненного цикла
      (сейчас `ConsoleMailer`, письма только логируются/пишутся в `_work/tmp/mail`).
- [ ] **Geocoder** — `GEOCODER=nominatim` + `NOMINATIM_URL`/`NOMINATIM_USER_AGENT` для реального
      геокодинга мест рождения на проде (сейчас `FixtureGeocoder`, известный набор городов).

## 4. SEO-запуск

- [ ] **Снять глобальный noindex**: задать `SEO_NOINDEX_ALL=false` в проде (дефолт — `true`,

      гейта (п. 1) — до этого момента сайт намеренно не индексируется (`robots.txt` отдаёт

- [ ] Настроить Яндекс.Метрику + цели воронки (калькулятор → регистрация → оплата) — счётчик и

- [ ] Сезонный контент: страницы «гороскоп 2027» — уже генерируются автоматически с октября (см.
