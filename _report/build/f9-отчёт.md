## 2. Что реализовано

**БД (миграция `0009_f9_sozvezdie_nebo_dnya.sql`, сид `0016_astra_user.sql`):**
`sky_days` (событие дня: payload + обезличенный транзитный снапшот SharePositions + пост-тред),
`sky_checkins` (отклик: вердикт hit/partial/miss + заметка→комментарий), `sky_streaks`
(стрик, правило — чистая `advanceStreak` в shared); `posts.author_kind`/`comments.author_kind`
('human'|'ai' — юридическая маркировка ИИ на уровне данных); `chart_shares.caption`;
`post_kind`+`sky_day`, `share_kind`+`transit_day`; системный пользователь «Астра»
(фиксированный UUID = `ASTRA_USER_ID`, невходимый пароль).

**API (`/api/v1/sky`):** `GET /today`, `GET /days/:dayKey` (публично, агрегаты чек-инов),
`GET /days/:dayKey/me` (проекция на натал: расшифровка self-профиля на лету, наружу только
аспекты — паттерн personal-horoscope), `POST /checkins` (только «сегодня» по МСК; заметка —
обычный комментарий треда через классификатор UGC; чек-ин+стрик — одна транзакция).
Share: `caption` у transit_day (в хэше слага, через `classifyUgcText`, 400 при флаге).
`GET /posts` исключает `sky_day` из общей ленты; `POST /posts` принимает только
`userPostKindSchema`.

**Worker:** кроны `sky-day-generate` (00:40 МСК, после astro-calendar/гороскопов — читает
свежие данные того же прогона; идемпотентный upsert «сегодня+завтра») и
`sky-astra-weekly-digest` (вс 18:00 МСК, детерминированный дайджест-комментарий).
Выбор события дня — чистая функция (ингресс → точная фаза → ретро → точнейший мажорный
аспект → Луна в знаке). Тексты дня — LLM+postprocess (stub-безопасно, fallback при флаге).
OG-карточка transit_day — компоновка 1200×630 (биколесо натал+небо + подпись отклика).

**Web:** публичные SSR-страницы `/nebo-dnya` (+архив `/nebo-dnya/@dayKey`), `/soobshchestvo`
(+`/soobshchestvo/@id`); старые `/app/soobshchestvo*` — тонкие редиректы; острова:
SkyPersonalBlock (проекция+чек-ин+стрик+share), ReactionBar (все 4 реакции наконец в UI),
CommentsBlock/PostView (общие для поста и треда дня); бейдж «ИИ» у контента Астры; NAV
+«Небо дня», FOOTER +«Сообщество»; `/podelitsya` понимает transit_day (подпись + CTA
«А как это небо у тебя?»).

**Найден и исправлен латентный баг ядра API:** `setErrorHandler` регистрировался ПОСЛЕ
роутов — дочерние контексты Fastify наследуют обработчик на момент своей регистрации,
поэтому ошибки валидации zod уходили в дефолтный обработчик, чей формат не проходил
сериализатор `apiErrorSchema` → клиент получал 500 `FST_ERR_FAILED_ERROR_SERIALIZATION`
вместо 400. Обработчик перенесён до регистрации роутов (см. doc-комментарий в app.ts);
найдено интеграционным тестом Ф9.

## 3. Верификация (integration/e2e — прогнано на dev-БД Supabase, ADR-8)

- Миграция 0009 + сиды применены (`pnpm db:migrate`, `pnpm db:seed`).
- `sky.integration.test.ts` — **5/5 зелёные**: публичный день → чек-ин с заметкой →
  комментарий в треде → агрегаты → стрик=1 → идемпотентность; запрещённая заметка →
  премодерация; `/me` без ПД; чек-ин «не сегодня» → 400; `POST /posts kind=sky_day` → 400;
  share transit_day roundtrip + флаг классификатора → 400.
- Живой стек (worker dist → api dist :3001 → web tsx :3000): генератор создал реальные дни
  («Новолуние в знаке Рака», сегодня+завтра, треды от Астры + дайджест); SSR `/nebo-dnya` —
  заголовок, колесо, canonical, JSON-LD, бейджи «ИИ», ноль утечек ПД; `/soobshchestvo`
  читается без логина; редирект со старого URL; архив 200 / несуществующий день 404;
  авторизованный цикл: проекция computed → чек-ин (стрик 1, агрегаты выросли) → заметка
  видна в треде → share-карточка с подписью открывается анонимом, ПД нет.

**Требует ручного шага (env/инфра):**
- OG PNG для transit_day: воркер+ObjectStorage (`STORAGE=s3`) — код есть, на stub-storage
  раздача PNG между процессами не проверяется (paths /og.png отвечает честным 404 до генерации).
- Полный интеграционный набор на удалённой Supabase упирается в лимит session-pooler
  (15 клиентов) и таймауты 5с — на локальном docker-Postgres ограничение не действует.
  Отмечен долг Ф7 (не блокер): `GET /posts` делает 3 параллельных запроса на пост (N+1).
- Реальный LLM-провайдер для текстов дня (`LLM_PROVIDER`) — сейчас stub-черновики с пометкой.
