## Устранённые находки аудита

**HIGH:** (1) утечка ПД — весь `charts.data` теперь шифруется AES-256-GCM (было: дата/время/
координаты рождения открытым текстом рядом с их шифрами); (2) `trustProxy:1` + Caddy XFF —
rate-limit по реальному клиенту; (3) 240 SEO-страниц `/planety/*` (были пустые+noindex на SSR) —
серверный рендер. **MED:** fail-fast COOKIE_SECRET; rate-limit LLM-эндпоинтов; вебхук ЮKassa
fail-closed; ленивая генерация лунного календаря; добавлен `/znaki-zodiaka/{znak}`. **LOW:**
pino-redaction синастрии; прегейт запрещённого во входе заказных прогнозов.

## Как запустить на дев-сервере

1. `pnpm install`
2. Заполнить `.env` из `.env.example` (`DATABASE_URL` на свой Postgres с pgvector; `STORAGE=s3` +
   `S3_*` для s3.cloud.ru; ключи по вкусу — иначе `*_PROVIDER=stub`).
3. Миграции: `pnpm --filter @stassist/db exec tsx tools/db-migrate.ts` (или см. README).
4. Сиды: применить `drizzle/seed/*.sql`; `pnpm data:horoscopes-backfill` для стартовых гороскопов.
5. Запуск процессов: `node dist/index.js` в `apps/{api,web,worker}` (после `pnpm -r build`).
6. Деплой на VPS: `docker compose` + Caddy (файлы готовы; нужен хост с docker и домен для TLS).
