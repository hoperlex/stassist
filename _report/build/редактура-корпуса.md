# Отчёт: редактура текстового корпуса «Зодиакум»

> Промт: `docs/roadmap/prompts/text-корпус-редактура.md`. Ветка `content/editorial` (от `main`).
> Метод: многоагентная редактура реальным Opus (Workflow: editor → verifier + дедуп-критик),
> творческий слой (агенты пишут JSON) отделён от детерминированной сборки SQL (скрипт).

## Итог одним взглядом

- **Переписано 1943 текстовые единицы** черновика (`draft`) → редакторский продовый уровень
  (`reviewed`/`published`) и доставлено **новыми надстроечными сидами `0012–0015`** (по требованию
  заказчика старые `0003/0004/0007/0008` НЕ переписывались).
- **Покрытие 100%**: chunks 1601/1601, compat 78/78, wiki 189/189, stones 75/75.
- **Проверка чистая**: 0 пустых, 0 дублей текста между ключами, 0 срабатываний forbidden-фильтра,
  0 клонов-зачинов.
- **Применение к живой БД проверено** (Postgres 16, scratch-БД): `db:migrate` + `db:seed` без ошибок,
  все строки `reviewed`, FK-линковка `0009` цела (1478 связей, 0 сирот), сиды идемпотентны.
- **Ворота собираемости зелёные**: `typecheck` / `lint` / `test:unit` / `build` — по всем пакетам.

## A. Рерайт корпуса (основной объём)

Инфраструктура (коммит «инфраструктура»): `packages/llm/src/corpus/manifest.ts`
(`buildEditorialManifest` — перечень 1601 ключа из `buildCorpus()`, без дрейфа; unit-тест),
`tools/gen-editorial-manifest.ts` (раскладка всего корпуса на 112 editor-батчей с брифами),
`tools/gen-corpus-reviewed.ts` (сборка `0012–0015`), `tools/check-reviewed-corpus.ts` (оффлайн-проверка).

Механика доставки: новые сиды применяются раннером `tools/db-seed.ts` ПОСЛЕ старых (readdir+sort),
upsert **без draft-guard** и с `quality/status='reviewed'` — перезаписывают draft, самозащитны
(старый `0003 WHERE quality='draft'` их больше не трогает), FK/линковку не ломают (ключи/slug/id и
`source_article_id` не меняются).

Многоагентный прогон: пилот (4 батча, валидация тона/качества) → волна 1 (68 батчей, 137 агентов)
→ волна 2 (40 батчей, 81 агент). Верификаторы на каждый батч ловили монотонность зачинов и правили;
финальный дедуп-критик устранил 6 кросс-батчевых клонов-зачинов (13 единиц). Токенов ≈ 8.3 млн.

| Сид | Таблица | reviewed | Разбивка |
|---|---|---:|---|
| `0012` | interpretation_chunks | **1601** | arcanum 660, aspect 425+обзоры 11, planet_in_sign 120, planet_in_house 120, numerology 88, point_in_sign 60, point_in_house 60, planet 14, sign 12, house 12, asc 12, element 4, cross 3 |
| `0013` | compat_pages | **78** | тексты пар знаков |
| `0014` | wiki_articles | **189** | glossary 57, lunar_days 30, nakshatras 27, arcana 22, planets 14, signs 12, houses 12, aspects 7, schools 6, stones 2 |
| `0015` | stones | **75** | properties/suitable/unsuitable (вычисленные массивы не тронуты) |

Правила качества соблюдены по построению промта редактора + проверкой верификатора: живой русский и
конкретика; анти-галлюцинации (без выдуманных дат/цифр/цитат); не-фатализм («склонность», не «приговор»);
запрет мед.диагнозов/темы смерти/гарантий/порчи (метафора 13-го Аркана допустима); уникальные зачины;
бренд «Зодиакум». Камни используют только факты `stones.json` + `docs/research/06`, дисклеймер сохранён;
накшатры — честный каркас (детали джйотиша не выдуманы, v1.x).

- **Глоссарий/вики** расширены качеством в рамках блока A (сид `0014`).

## C. Прокачка рантайм-генераторов (промт-инжиниринг, не ручной текст)

- `packages/llm/src/postprocess/forbidden-filters.ts`: **учёт отрицания** (не рубит «не ставлю
  диагнозы», «ничего не гарантирует») + фикс over-match (`(?<!в)лечени` — не ловит «влечение»;
  `болезн(?!енн)` — не ловит метафору «болезненно»; `(?<![а-яё])точно ли` — не ловит «достаточно ли»).
  Реальные запросы («поставь диагноз», «сниму порчу», «гарантируете замужество») по-прежнему ловятся.
- `packages/llm/src/prompt/system-rules.ts`: + конкретика (привязка к факту из ДАННЫХ), «как считали»,
  финальный дисклеймер один раз. `PROMPT_VERSION` `f4-2026.07.1 → .2` (осознанная инвалидация cache_key).
- `packages/llm/src/horoscope/{tone,build-prompt,templates}.ts`: анти-штамп/конкретика в системных
  правилах, антидубляж зачинов между знаками выпуска, расширены пулы зачинов/советов/закрывающих фраз
  детерминированного stub-пути (инварианты длины сохранены).
- **Тесты**: `postprocess/forbidden-filters.test.ts` (recall + негатив + over-match), `horoscope/tone.test.ts`,
  `horoscope/build-prompt.test.ts`; расширены существующие. `@stassist/llm` — 489 unit-тестов зелёные.

## Верификация

1. **Гейт собираемости** — зелёный по всем пакетам: `pnpm -r typecheck && pnpm -r lint &&
   pnpm -r test:unit && pnpm -r build`. (В общем `-r`-прогоне возможны редкие таймауты из-за
   конкуренции за CPU — каждый пакет по отдельности зелёный.)
2. **Оффлайн-проверка корпуса** (`pnpm data:check-reviewed`): 1943/1943 покрытие; 0 пустых; 0 дублей
   текста между ключами; 0 повторяющихся зачинов; `detectForbidden` = 0 срабатываний.
3. **Применение к живой БД** (scratch Postgres 16, DATABASE_URL): `db:migrate` + `db:seed` без ошибок;
   1601 chunk = `reviewed`/`version=2`/0 draft; 78 compat с body_md; 189 wiki + 75 stones `reviewed`;
   0 пустых, 0 дублей текста; FK `source_article_id` — 1478 связей, 0 сирот; повторный `db:seed`
   идемпотентен (1601 reviewed / 0 draft).

## Что осталось (требует ручного шага)

- **Реэмбеддинг корпуса**: тексты изменились → `interpretation_chunks.embedding` (в сидах `NULL`)
  неактуальны для семантического слоя. Key-based ретрив не страдает. Backfill — интеграционный job
  (`apps/worker`, `EMBED_PROVIDER≠stub`).
- **Прогон гороскопов/разборов реальным LLM** (блок C) — задача рантайма/заказчика; здесь улучшены
  промты и тесты на них.
- **Накшатры (27)** — честный каркас; детальные трактовки джйотиша — контент v1.x (не выдумываем).
- Применение `pnpm db:seed` к проду — ручной шаг (требует `DATABASE_URL`).
