## 2. Что реализовано

### 2.1 Миграции (см. находку [порядок-миграций/FK])

`drizzle/migrations/0004_llm_pipeline.sql` (сгенерирован `drizzle-kit generate`, офлайн, из
`packages/db/src/schema/{wiki-articles,interpretation-chunks,ai-reports,chat,report-feedback}.ts`):

- `wiki_articles` — создана **ПУСТОЙ** (skeleton) вместе с `interpretation_chunks` в этой же
  миграции — FK `interpretation_chunks.source_article_id → wiki_articles.id` валиден уже в Ф4.
  Наполнение — Ф7 (как и зафиксировано в §5 конвенций).
- `interpretation_chunks` — `key` uniq, `tradition` enum, `text`, `source_article_id` fk
  nullable, `embedding vector(1024)`, `quality` enum(draft/reviewed), `version`. Индекс
  `interpretation_chunks_embedding_hnsw_idx` — `USING hnsw ("embedding" vector_cosine_ops)`.
- `ai_reports` — по доку 22 §3 + доп. поля аудита: `corpus_version`, `chunks_used jsonb` (аудит
  использованных ключей+версий чанков, находка [внутренняя-полнота-аудит]), `error_message`,
  `completed_at`. Индекс `ai_reports_cache_key_idx` (не уникальный — историчность).
- `chat_sessions`/`chat_messages`, `report_feedback` — по доку 22 §3 буквально.

`pnpm db:generate` после реализации — «No schema changes, nothing to migrate» (схема стабильна).

### 2.2 `packages/llm` (полностью с нуля, было пустой заглушкой Ф0)

- **Сериализатор фактов** (`facts/serializer.ts`): `ChartData → {text, factKeys}`,
  детерминированный (snapshot-тест). Считает то, чего НЕТ в `ChartData` от astro-core: 7
  эссенциальных достоинств (классическая таблица 7 планет — `facts/dignities.ts`), стеллиумы
  (`facts/stelliums.ts`), Южный узел (Северный+180°, включая дом через куспиды). Обрабатывает
  `meta.noHouses` — не упоминает дома/Асцендент вообще, если время неизвестно.
- **Ключи корпуса** (`facts/keys.ts`) — единственный источник правды по формату
  `interpretation_chunks.key`, используется И генератором корпуса, И сериализатором/ретривером —
  формат не может разойтись. Явно задокументированы и разрешены все неоднозначности источников
  (EN-слаги знаков, «планеты(14)», «аспекты(7)» → 11, канон Южного узла и т.д.).
- **Провайдеры**: `AnthropicLlmProvider` (приоритет, `claude-sonnet-5`), `OpenRouterLlmProvider`,
  `YandexGptLlmProvider`, `GigaChatLlmProvider` — fetch-based, без SDK-зависимостей, таймаут+
  инъекция `fetchImpl` для тестов. `FallbackLlmProvider` — цепочка по `LLM_FALLBACK_CHAIN`,
  ретраи с джиттером (`providers/retry.ts`), circuit breaker (`providers/circuit-breaker.ts`).
  `CohereEmbeddingProvider` — единственный поддерживаемый «настоящий» embed-провайдер
  (embed-multilingual-v3.0, ровно 1024-dim, RU-совместим).
- **RAG**: `rag/chunk-repository.ts` — порт (`packages/llm` НЕ зависит от `@stassist/db`),
  `InMemoryChunkRepository` для тестов; `rag/retriever.ts` — точный ретрив по ключам + откат
  `aspect:*` на `aspect:<angle>:overview`, если точного чанка нет; `missingKeys` — честно, не
  выдумываем. Семантический ретрив (`retrieveSemantic`) — опционален, для чата.
- **Промт**: `prompt/system-rules.ts` (все «незыблемые правила» текстом), `prompt/build-prompt.ts`
  (данные + чанки + задача + структура).
- **Пост-обработка**: `postprocess/forbidden-filters.ts` (regex-слой: death/medical/curse/
  guarantee, ложно НЕ триггерится на легитимном упоминании Аркана «Смерть»), `postprocess/
  structure.ts`, `postprocess/disclaimer.ts` (переиспользует `DISCLAIMER_TEXT_RU`), `postprocess/
  postprocess.ts` — оркестрация → `flagged`+`flagReasons`.
- **Кэш**: `cache/cache-key.ts` — см. §5.
- **Отчёты**: `reports/report-kinds.ts` (тариф+синхронность по типу), `reports/time-unknown.ts`
  (см. §4), `reports/pipeline.ts` — `generateReport()`, полная оркестрация без БД/сети (принимает
  `LlmProvider`/`ChunkRepository` как параметры).
- **Чат**: `chat/intent-router.ts` (natal/transit/synastry по ключевым словам + forbidden),
  `chat/chat-pipeline.ts` — `answerChatQuestion()`.
- **Корпус**: `corpus/*` — см. §3.

### 2.3 Воркер/API-слой

- `apps/worker/src/llm/{drizzle-chunk-repository,charts-repository,ai-reports-repository,
  generate-report-job}.ts` + `worker.ts`: очередь `ai-report-generate`, cron `*/1 * * * *` —
  сканирует `ai_reports.status='queued'`, генерирует, сохраняет (`done`/`flagged`/`failed`).
- `apps/api/src/routes/ai-reports.ts`: `POST /api/v1/ai-reports` — `big3` синхронно (см. §4),
  остальные — `queued` + 202; кэш-хит по `cache_key` — не платит повторно; premium-типы — 402
  пейвол-заглушка. `GET /api/v1/ai-reports/:id` — статус-поллинг.
- `apps/api/src/routes/interpretation.ts`: `GET /api/v1/calc/interpretation?keys=...` —
  публичное (без авторизации) пакетное чтение корпуса — см. §6, п.8.

**«Нет пустых ключей-потребителей Ф3/Ф5/Ф6/Ф7» — подтверждено структурно, не выборочно.**
Генератор строит корпус ПОЛНЫМ декартовым перебором (циклы по каноническим спискам из
`facts/keys.ts`/`facts/matrix-positions.ts`/`facts/numerology-positions.ts`), поэтому пропуск
структурно невозможен — это проверяет `packages/llm/src/corpus/build-corpus.test.ts` (14 тестов,
часть общего build-гейта, а не отдельный ad-hoc скрипт): полное покрытие знаков/планета×знак/
планета×дом/кармических точек/всех 11 аспектов/всех пар классических планет/всех 22×30 позиций
арканов/примеров ключей из документации (`numerology:life_path:7`, `numerology:matrix_cell:health`,
`aspect:sun:square:moon`) + отсутствие дублей + непустота каждого текста + 78 пар совместимости.

**Эмбеддинги НЕ включены в committed seed** (текст-only, ~1.3 МБ на 1601 строку) — вычисление
1024-мерных векторов для всего корпуса требует реального `EMBED_PROVIDER=cohere` (интеграционный
шаг, не оффлайн data-step по духу §4 конвенций: коммитить мегабайты векторных литералов
нецелесообразно). Ретрив по `key` при этом работает **полностью офлайн** — `embedding` в схеме
nullable, HNSW-индекс валиден и на пустой/частично заполненной колонке.

## 4. Как решены критичные требования задания

- **time_unknown** (`reports/time-unknown.ts`, находка [внутренняя-полнота-edge-case]):
  деградация, НЕ отказ. `big3` при `noHouses=true` → Солнце+Луна с явной пометкой в промте
  («Асцендент не рассчитан»), сериализатор физически не выдаёт домов/Асцендента в блок ДАННЫЕ
  (сам факт-ключ `asc_in_sign:*` не строится). `natal_full` — для сфер `relationships/career/
  resources` (завязаны на дома) добавляется явная пометка об ограничении; сериализатор всё равно
  не упоминает дома. Проверено юнитами (`serializer.test.ts`, `pipeline.test.ts`).
- **cache_key** (`cache/cache-key.ts`, находка [корректность-кэша]): sha256 от
  `birthProfileId|kind|promptVersion|presetId|coreVersion|sphere|partnerBirthProfileId|question`.
  Смена пресета ИЛИ бамп `core_version`/`PROMPT_VERSION` → новый ключ (протестировано явно).
  `apps/api/src/routes/ai-reports.ts` ищет `status='done'` по этому ключу ПЕРЕД генерацией.
- **draft-публикация** (§6 конвенций, находка [публикация-до-готовности]): весь seed-корпус —
  `quality='draft'`. Ретривер (`rag/chunk-repository.ts`/`retriever.ts`) НЕ фильтрует по quality
  вообще — draft и reviewed отдаются на равных, `quality` прокидывается наружу
  (`interpretationChunkResponseSchema`), UI (`InterpretationBlock.tsx`) показывает бейдж
  «черновик» — draft НЕ скрывается, ровно по правилу непустоты.
- **эмбеддинги** (находка [контракт-эмбеддингов]/[внутренняя-полнота-эмбеддинги]): единственный
  «настоящий» embed-провайдер — Cohere `embed-multilingual-v3.0`, ровно 1024-dim (валидируется в
  рантайме — бросает, если провайдер вернул другую размерность), метрика — cosine
  (`vector_cosine_ops`, HNSW). `LLM_PROVIDER` и `EMBED_PROVIDER` конфигурируются НЕЗАВИСИМО.
  Anthropic как embed-провайдер явно НЕ реализован (`createEmbeddingProvider` бросает понятную
  ошибку) — ретрив по `key` в принципе не требует эмбеддингов.
- **анти-галлюцинации**: системный промт (`prompt/system-rules.ts`) — «позиции только из блока
  ДАННЫЕ»; сериализатор — единственный источник чисел в промте; пост-фильтры
  (`postprocess/forbidden-filters.ts`) — regex-слой на всех провокационных кейсах из раздела
  «Верификация» промта (когда я умру / сниму ли порчу / медицинские вопросы) — проверено
  `it.each` по всем перечисленным кейсам, включая non-regression на легитимном контенте (Аркан
  «Смерть»); автодисклеймер добавляется всегда (`postprocess/disclaimer.ts`); аудит —
  `ai_reports.chunks_used`/`calc_block`/`prompt_version`/`corpus_version`/`provider`/`tokens_*`/
  `cost_micros`.
- **модель исполнения**: `big3` — синхронно в `apps/api` (см. `report-kinds.ts
  allowSynchronous`); остальные — `ai_reports.status: queued→generating→done|failed|flagged`,
  воркер-cron `*/1 * * * *` (тот же паттерн, что `chart-share-og-backfill` из Ф3 — API не держит
  соединения к pg-boss). Клиент наблюдает статус поллингом `GET /:id` (SSE — не в скоупе Ф4).

## 5. Верификация — что сделано (юнит) и что integration/«требует ручного шага»

**Юнит (часть гейта, все зелёные, без сети/БД):**
- Сериализатор — снапшот + 8 явных юнитов (в т.ч. канонический порядок аспектной пары,
  дом Южного узла через куспиды, стеллиумы, эссенциальные достоинства, time_unknown).
- Ретривер — точные ключи без эмбеддингов, откат аспектов, «не выдумываем» (missingKeys).
- Фильтры запрещённого — все провокационные кейсы из промта + non-regression на арканах.
- Failover — `fallback.test.ts`: отключение первого провайдера → второй подхватывает
  (circuit breaker открывается после N неудач, повторный вызов сразу идёт ко второму).
- Кэш — `cache-key.test.ts` (детерминизм/чувствительность к пресету/ядру/промту).
- Генератор корпуса — детерминизм + полное структурное покрытие (замена «10 карт вручную»).
- Конвейер отчёта — `pipeline.test.ts` (в т.ч. StubLlmProvider end-to-end, time_unknown,
  provocative-flagged).

**Integration/«требует ручного шага» (не в юнит-гейте, гейтится наличием инфры/ключей):**
- `apps/api/src/routes/ai-reports.integration.test.ts` — требует `DATABASE_URL` (авто-skip):
  полный цикл big3 синхронно + кэш-хит + 402-пейвол для `natal_full`. Требует, чтобы миграции
  были применены и `pnpm data:seed-calc-presets && pnpm data:corpus && pnpm db:seed` выполнены.
- Реальные вызовы `AnthropicLlmProvider`/`OpenRouterLlmProvider`/`YandexGptLlmProvider`/
  `GigaChatLlmProvider`/`CohereEmbeddingProvider` — HTTP-контракты реализованы по публичной
  документации, но НЕ проверены живым ключом (нет доступа в среде сборки) — требует
  `ANTHROPIC_API_KEY`/`OPENROUTER_API_KEY`/`YANDEXGPT_API_KEY`+`YANDEXGPT_FOLDER_ID`/
  `GIGACHAT_API_KEY` (уже полученный OAuth access-токен — автоматический обмен client_id/
  client_secret на токен НЕ реализован в Ф4, отмечено в коде явно) / `COHERE_API_KEY`.
- Семантический (pgvector) ретрив — код есть (`retrieveSemantic`, `DrizzleChunkRepository.
  semanticSearch`), но некому проверить без живого Postgres+pgvector И реально заполненных
  эмбеддингов (см. §3 — они не в committed seed).
- Панель качества (20 карт, LLM-judge) и golden-набор эталонных разборов — **не реализованы**:
  вне списка «Что построить» текущего задания; для проведения нужен живой LLM-ключ и ручная
  редакторская панель — рекомендовано отдельным шагом Ф4+/Ф6.
- Генерация PDF (`ai_reports.pdf_key`) — **не реализована** (поле в схеме зарезервировано,
  nullable) — вне списка «Что построить» текущего задания; исходный промт Ф4 (req.7) относит её
  к отдельному worker-шаблону (playwright-chromium), рекомендовано Ф6 (там же PDF «Матрица
