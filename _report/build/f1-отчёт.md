## 2. Что реализовано

### 2.1 `packages/shared` (аддитивно)
`src/schemas/chart.ts` — `ChartInput`/`ChartData` и все связанные zod-схемы (единый источник
правды для API/фронта/ядра, см. 21 §3.2). `bodies`/`points` — `z.object` с опциональными полями
(а не `z.record` по enum-ключу — иначе TS считает все ключи обязательными, что неверно для
взаимоисключающих истинный/средний узел, опционального Хирона и т.д.).

### 2.2 `packages/astro-core`

| Слой | Модуль | Статус |
|---|---|---|
| Время | `time/{julian-day,timezone,sidereal-time}.ts` | JD UT/TT через `astronomy-engine` (ΔT Espenak–Meeus, единый источник для всего ядра); IANA tz → UTC через `Intl`/ICU (без доп. зависимостей, историческая LMT-эпоха работает); GMST (Меёс)/GAST (+ уравнение равноденствий из `e_tilt`) |
| Позиции | `ephemeris/{sun-vsop87,planets,nodes-lilith,chiron}.ts` | Меркурий–Плутон+Луна — `astronomy-engine` (`GeoVector`+`Ecliptic`, видимая эклиптика даты); Солнце — независимый модуль VSOP87D (реальные официальные данные IMCCE, TS-таблица сгенерирована `tools/gen-vsop87.ts`), **единый движок натал+соляр**; средний/истинный узел (истинный — аналитически из осевых элементов, численно проверен); средняя Лилит/Селена (полиномы Меёса, проверены на устойчивость невязки); истинная Лилит — `NotImplementedError` (версия 2+, задокументировано); Хирон — реальные данные JPL Horizons → кусочный Чебышёв |
| Дома | `houses/*.ts` | Плацидус (итеративно, выведен от первых принципов), Регимонтан (векторное построение, численно проверено), Кох (честно промаркированное приближение — см. ACCURACY.md), Порфирий/равнодомная/целознаковая (точно). Fallback >66.5° → Порфирий |
| Аспекты | `aspects/*.ts` | Мажор+минор, конфигурируемые орбисы по аспекту/телу, аппликация/сепарация по знаку относительной скорости |
| Root-finding | `rootfinding/*.ts` | Брент (реализован и проверен на x²−2), ингрессы, точные транзитные аспекты, void-of-course, соляр/лунар (единый движок Солнца/Луны) |
| Прогностика 6б | `forecast/*.ts` | Вторичные прогрессии («день за год»), символические дирекции (+k°/год), композит (короткая дуга), Davison (середина времени/места) — все с явными числовыми инвариантами в тестах |
| Арабские части | `arabic-parts/fortuna.ts` | Фортуна день/ночь по секте + каркас `computeArabicPart` для расширения |
| Сидерика/джйотиш | `sidereal/*.ts` | Айанамши (приближённо, см. ACCURACY.md), накшатры, вимшоттари-даша — **не выведены в `ChartData`**, «про запас» |
| Лунный календарь | `lunar-calendar/*.ts` | Фазы (элонгация Солнце-Луна + обёртка над `SearchMoonQuarter`), лунные дни по восходам (`SearchRiseSet`) |
| Ба-цзы | `bazi/index.ts` | 4 столпа, стволы/ветви/стихии — с явно помеченными допущениями (см. ACCURACY.md) |
| Оркестрация | `chart.ts` | `computeChart(input): ChartData` — единственная точка входа, чистая функция |

### 2.3 `packages/numerology-core` (реализован параллельным агентом, независимо перепроверен)
Число жизненного пути, числа по имени (кириллица, циклическая таблица), психоматрица Пифагора,
персональные циклы, совместимость по числам, Матрица судьбы (9 базовых точек октаграммы —
`methodologyVerified: true`; 7 чакр/4 предназначения/линии/возрастные периоды —
`methodologyVerified: false`, честно помечено). См. `packages/numerology-core/README.md`.

## 3. Ворота Ф1 (§7 конвенций) — исполнены

1. **Human-verified seed** — 5 карт, `packages/astro-core/fixtures/golden/*.json`. Долготы
   Солнца/Луны/8 планет — **реальные значения JPL Horizons API** (живой запрос при сборке,
   `QUANTITIES=31`, апарентная геоцентрическая эклиптика даты). Углы/дома — регрессия против
   собственных формул, которые отдельно численно проверены через матрицы поворота
   `astronomy-engine` (см. ACCURACY.md §3.1 — разграничение честно отражено в `source` каждой
   фикстуры). Допуски: планеты ≤1′, куспиды/углы ≤2′ — **все 60 проверок проходят**.
2. **Кросс-валидация** против `ephemeris` (npm, Moshier/DE404, независимый источник, devDependency,
   не в рантайме) — 1900–2050, случайные даты. Максимум расхождения за прогон: Солнце ~4–7″,
   Луна ~50–60″, планеты ~4–18″ — все в допуске (планеты/Солнце ≤2′, Луна ≤3′).
3. **Property-тесты** — непрерывность долгот, ретро⇔скорость, сумма домов=360°, симметрия
   аспектов/композита — зелёные.

**Максимальное отклонение кросс-валидации за представленный прогон: ~60″ (Луна), ~18″ (прочие
тела)** — печатается в консоль при каждом запуске `test:unit` (числа стабильно в допуске, но
плавают на единицы″ между прогонами из-за случайной выборки дат — см. точные цифры конкретного
прогона выше в §1/в выводе `pnpm -r test:unit`).

## 4. Оффлайн data-steps (§4 конвенций) — НЕ часть build/CI

- `pnpm data:gen-vsop87` (`tools/gen-vsop87.ts`) — парсит закоммиченный кэш
  `tools/data/vsop87d-earth-raw.txt` (реальные официальные данные VSOP87D, IMCCE) →
  `packages/astro-core/src/ephemeris/vsop87-earth-data.generated.ts` (274 члена после усечения
  по амплитуде).
- `pnpm data:gen-chiron` (`tools/gen-chiron.ts`) — тянет реальные данные JPL Horizons (кэш
  закоммичен в `tools/data/chiron-horizons/*.txt`, 40 чанков, 36 560 точек, 1900–2100 шаг 2 сут)
  → кусочный Чебышёв (200 окон по году, степень 10–14 адаптивно; макс. невязка фита по всем
  окнам 27.97″, средняя 7.79″) → `packages/astro-core/src/ephemeris/chiron-data.generated.ts`.

Оба скрипта — кэш-first (повторный прогон без сети, если кэш уже закоммичен); ни один не входит
в `build`/`test:unit`/CI-гейт.

Полный расчёт натала (10 планет+Луна+Хирон+узел+Лилит+Селена+Плацидус+аспекты+Фортуна):
**<50мс** тёплый прогон. Соляр (root-finding момента возврата Солнца): **<200мс**. Оба —
юнит-тесты с явным бюджетом (`index.test.ts`, `rootfinding/rootfinding.test.ts`).
