# Находки верификации для F7

> Учитывай при реализации. Сквозные системные правила — в `docs/roadmap/31-конвенции-реализации.md` (приоритетны при конфликте с промтом).

## MAJOR (5)
- **[content-provenance]** Наполнение базы celebrities (100 персон) не имеет источника данных — риск точности и юридики
  - _fix:_ Указать источник данных и критерий приёмки: брать только записи с Родден-рейтингом AA/A/B из открытого каталога (Astro-Databank/Wikidata), для каждой персоны сохранять source_url и rodden в birth_data; добавить в «Верификацию» пункт «celebrities: 100 записей, у каждой заполнены source и rodden∈{AA,A,B}, время неизвестно → time_unknown=true». Либо сократить стартовый объём и предоставить сид-CSV.
- **[data-model-gap]** Модель хранения анонимизированной копии карты не определена и конфликтует со схемой charts
  - _fix:_ Явно задать хранение анонимной копии (напр.: новая charts-строка kind='anon_review' с birth_profile_id=NULL — тогда отметить поле nullable в 22 §2 — либо поле posts.anon_chart jsonb) и перечислить точный набор допустимых полей ПД-free-JSON (planets[], houses[], aspects[]; запрещены birth_date/birth_time/place/lat/lon/name), по которому пишется автотест.
- **[missing-step]** Правило привязки interpretation_chunks.source_article_id к статьям не определено, хотя это DoD
  - _fix:_ Задать однозначное правило маппинга (напр.: planet_in_sign/planet_in_house → статья соответствующей планеты; aspect → статья аспекта; arcanum → статья аркана) и явно поручить Ф7 бэкфилл source_article_id при публикации статей; уточнить, что программатические 240 страниц не создают wiki_articles, а чанки линкуются к разделу-планете.
- **[публикация-до-готовности]** Не определён режим публикации draft-чанков: Ф4 генерирует корпус со status='draft' (reviewed ставит редактор вручную), а Ф7 строит 240 SSR-страниц /planety/{planeta}-v-{znak|dom} «из чанков Ф4» — если на SSR идут только reviewed, страницы будут пустыми до ручной вычитки ~500+ чанков.
  - _fix:_ Явно прописать политику: для программатических вики-страниц допускается публикация чанков со статусом draft с пометкой «черновик/на редактуре» (или ввести промежуточный статус auto_approved после второго LLM-прохода + фильтров), а reviewed — приоритет ранжирования. Указать это в f4 (req.3) и f7 (req.2). Добавить в верификацию Ф7 проверку «доля непустых страниц кластера = 100%».
- **[empty-states-не-заданы]** Пустые состояния (empty states) страниц не специфицированы ни в одном промте: кабинет (пустой список профилей, нет отчётов, пустой чат, нет заказов), лента коммьюнити, календарь до backfill — нет ни текстов, ни требований.
  - _fix:_ Добавить сквозное требование (в план 30 и в промты Ф2/Ф4/Ф6/Ф7): для каждого списка/ленты/кабинетной страницы обязателен empty-state с осмысленным русским текстом и CTA (напр. «У вас пока нет профилей рождения — создайте первый»). Включить empty-states в чек-лист верификации фаз.

Дать хотя бы 1 эталонный шаблон статьи (разделы + обязательные факт-поля).
- **[ambiguity]** Состав «стартового набора» вики для DoD противоречив
  - _fix:_ Дать один явный перечень стартового набора с числами (какие разделы и по сколько статей обязательны для DoD MVP), устранив расхождение между M7.1, M7.2 и разделом «Верификация».
- **[missing-step]** Расширение pg_trgm и триграммный индекс для поиска по вики не оговорены
  - _fix:_ Добавить требование миграции: CREATE EXTENSION IF NOT EXISTS pg_trgm и GIN-индекс (gin_trgm_ops) по полям поиска wiki_articles; указать, ищем ли по title+body_md.
- **[data-model-gap]** Взаимность согласия на карту (chart_shared) для синастрии не определена — риск неверной приватной модели
  - _fix:_ Уточнить семантику: дружба — две направленные строки, chart_shared означает «владелец user_id открыл свою карту для friend_id»; виджет синастрии доступен только когда обе стороны chart_shared=true; отзыв любой стороны немедленно закрывает виджет. Добавить в DoD кейс односторонней расшаренности → 403/виджет недоступен.
- **[страница-без-владельца]** Отдельные SEO-страницы `/arkan/{1..22}` (матрица 23 §2, 23 URL) не назначены явно ни одному промту: арканные значения есть внутри `/matrica-sudby` (Ф3) и в вики-арканах (Ф7), но standalone /arkan/{n} нигде не строится.
  - _fix:_ Явно закрепить `/arkan/{n}` за фазой (логично Ф6 вместе с арканными PDF или Ф7-вики как алиас), из арканных чанков Ф4; либо задокументировать, что /arkan/{n} = редирект/алиас на /wiki/arcana/{slug}, чтобы не потерять кластер.
- **[рефит-плейсхолдеров]** Краткие текстовые блоки публичной страницы /natalnaya-karta остаются заглушками навсегда: Ф3 явно ставит плейсхолдеры «из вики-чанков», но ни Ф4, ни Ф7 не содержат шага замены их на реальный корпус.
  - _fix:_ Добавить в Ф4 (или Ф7) явную задачу: подключить краткие текстовые блоки /natalnaya-karta к чанкам planet_in_sign/planet_in_house/Asc корпуса и убрать заглушки; включить в верификацию проверку непустых блоков.
- **[community-anon-unit]** Ф7: критичный тест анонимизации карты (отсутствие ПД) описан только как E2E/по БД, хотя это чистая функция над схемой shared.
  - _fix:_ Вынести анонимайзер в чистую функцию и покрыть unit-тестом по zod-схеме (нет полей даты/времени/места/имени) без БД. pg_trgm-поиск, лента, LLM-модерация — интеграция/гейт.
