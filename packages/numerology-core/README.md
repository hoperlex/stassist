# @stassist/numerology-core

Детерминированные TS-функции нумерологии портала «Зодиакум»: без I/O (`fs`/`fetch`/`Date.now()`
внутри логики не используются — дата/«текущая» дата всегда передаются параметром).

## Состав пакета

- `src/constants.ts` — таблица «буква → число» (кириллица), список гласных.
- `src/text-utils.ts` — нормализация строки имени.
- `src/date-utils.ts` — чистые календарные утилиты (без системных часов).
- `src/digit-utils.ts` — примитивы редукции цифр (общие для всех модулей).
- `src/core-numbers.ts` — число жизненного пути (ЧЖП), числа по имени (выражение/душа/личность).
- `src/psycho-matrix.ts` — психоматрица (квадрат Пифагора).
- `src/personal-cycles.ts` — персональные год/месяц/день.
- `src/compatibility.ts` — совместимость по ЧЖП (таблица 9×9).
- `src/matrix-of-destiny.ts` — матрица судьбы (метод «октаграммы»).
- `src/schemas.ts` — zod-схемы входных/выходных данных, локальные для пакета.

## Осознанные методические решения

### Число жизненного пути (ЧЖП)

Используется метод **«сквозного сложения всех цифр»** даты рождения (день+месяц+год как единая
строка цифр), а НЕ «метод трёх циклов» (день/месяц/год редуцируются раздельно, затем
складываются). Оба метода общеприняты в рунете, но дают разные мастер-числа на части дат — портал
фиксирует один вариант, см. docs/research/06-нумерология-и-минералы.md §1. Мастер-числа **11 и
22** не редуцируются дальше; 33 мастер-числом НЕ считается (редуцируется как обычное число).

### Числа по имени (кириллица, пифагорейская система)

Таблица «буква → число» — **циклическая 1–9** (значение = порядковый номер буквы в алфавите по
модулю 9), вынесена в константу `LETTER_VALUES` (`src/constants.ts`) — единого стандарта для
кириллицы не существует, редактору достаточно скорректировать одну константу, не трогая логику.

Решения по спорным буквам:
- **Ё** — отдельная буква (не приравнивается к Е), получает следующее по циклу значение (7).
- **Й** — считается согласной, участвует в общем цикле наравне с прочими буквами.
- **Ъ, Ь** — не гласные (попадают в «согласные»/число личности), но участвуют в общем цикле как
  полноценные буквы алфавита — техническое решение для детерминированности расчёта.

Гласные для числа души (Soul Urge) зафиксированы явно: **А Е Ё И О У Ы Э Ю Я**.

Мастер-числа 11/22 сохраняются и в числах по имени (по аналогии с ЧЖП) — самостоятельное решение
портала для целостности продукта, отдельно не сверялось с конкретным эталонным сервисом.

### Психоматрица (квадрат Пифагора)

4 дополнительных числа считаются по стандартной методике (см. docs/research/06 §1): (1) — сумма
всех цифр даты; (2) — сумма цифр числа (1) за один проход (без принудительной редукции до одной
цифры); (3) — (1) минус удвоенная первая цифра дня рождения; (4) — сумма цифр числа (3) за один
проход. Мультимножество для сетки = все цифры даты + все цифры чисел (1)–(4), **нули не
учитываются** (стандартное правило метода).

Раскладка сетки 3×3 — традиционный квадрат Пифагора, читается по столбцам сверху вниз:
характер(1,2,3) / энергия(4,5,6) / интерес(7,8,9). Модуль возвращает только счётчики и суммы
(строки/столбцы/диагонали) — текстовые трактовки ячеек в пакет намеренно не включены (это уровень
контента, не детерминированного ядра).

### Совместимость по ЧЖП

Таблица совместимости 9×9 построена по прозрачной формуле «циклическое расстояние между базовыми
числами на круге из 9 позиций» (0 → 100 баллов, …, 4 → 40 баллов). Мастер-числа 11/22 для целей
ЭТОЙ таблицы приводятся к базовым 2 и 4. **Это РАБОЧАЯ ЭВРИСТИКА, требующая контентной сверки
редактором**, а не выверенный факт — конкретные оценки совместимости не подтверждены внешним
источником.

### Матрица судьбы

Метод «октаграммы» (нумерология + 22 старших аркана Таро), см. docs/research/06 §2. Редукция чисел
матрицы (`reduceToArcanum`) — отдельное от ЧЖП правило: любое число >22 сводится повторной суммой
цифр, пока не станет ≤22 (диапазон 1–22, индексы арканов Таро; мастер-чисел здесь нет). Граничное
правило: результат 0 заменяется на 22.

**9 базовых точек октаграммы** (`day`, `month`, `yearSum`, `tasks`(D), `center`(E),
`squareVertex1..4` / алиасы `f1..f4`) — общепринятая часть метода, одинакова во всех известных
клонах-калькуляторах (matricaladini.ru, matrica-sudby.ru, tvoyamatritsa.ru и др.).
`corePointsMethodologyVerified: true`.

## Известные ограничения

**Производные секции матрицы судьбы** (`derivedSections`: 7 чакр, 4 предназначения, линия
отношений, денежная линия, возрастные периоды) считаются у разных сервисов ПО-РАЗНОМУ — единого
стандарта нет. Поле `methodologyVerified: false` во всех результатах этой части.

> МЕТОДИКА ТРЕБУЕТ СВЕРКИ ЗАКАЗЧИКОМ (напр. с matricaladini.ru) — представленная формула является
> рабочей гипотезой по общепринятой схеме «матрицы судьбы», НЕ сверена с конкретным эталонным
> сервисом.

Конкретные формулы (осознанный выбор одной из правдоподобных схем, не единственно верный):
- 7 чакр = `reduceToArcanum(center + периферийная точка)` для точек `day, month, yearSum, tasks,
  f1, f2, f3` (7 из 8 периферийных точек, по порядку);
- 4 предназначения: личное = `day+f1`, родовое = `month+f2`, духовное = `yearSum+f3`,
  социальное/планетарное = `tasks+f4` (каждая сумма редуцируется);
- линия отношений = `reduceToArcanum(f1+f3)`, денежная линия = `reduceToArcanum(f2+f4)`
  (диагонали прямого квадрата);
- возрастные периоды — 8 периодов по 9 лет (0–72 года), период `i` привязан к `i`-й периферийной
  точке в порядке `day, month, yearSum, tasks, f1, f2, f3, f4`.

**Таблица совместимости по ЧЖП** (`compatibility.ts`) — числовые оценки 0–100 построены формулой
(«циклическое расстояние»), это рабочая эвристика, требующая контентной сверки редактором, а не
сверенный факт.

**Числа по имени**: сохранение мастер-чисел 11/22 — самостоятельное решение портала (по аналогии с
ЧЖП), отдельно не сверялось с внешним эталонным источником.
