# Портал Stassist — правила репозитория

Этот файл **дополняет** правила рабочего пространства `../CLAUDE.md`. Здесь — конвенции,
специфичные для git-репозитория `stassist` (origin `github.com/hoperlex/stassist`).

## Git

Репозиторий ведём напрямую в ветку `main` (без feature/topic-веток).

- **Изолированные коммиты.** Один логический блок изменений — один коммит; несвязанные правки
  в один коммит не смешиваем. Стейджим файлы по явным путям (`git add <путь>`), а не `git add -A`,
  чтобы не захватить чужие/несвязанные изменения рабочего дерева.
- **Автокоммит и пуш.** После выполнения задачи и прохождения проверок (typecheck / lint /
  верификация) сразу делаем коммит и `git push` в `origin/main` — без отдельного запроса.
- Перед пушем — секрет-проверка: реальные значения из `.env` не должны попасть в git
  (`.env` под git-ignore).
- Сообщения коммитов — на русском.
