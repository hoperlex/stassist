# Промт Ф5 — Гороскопный пайплайн, программатика, шуточный контур

## Контекст
Фаза Ф5 портала «Зодиакум». Прочитай: `docs/architecture/20-архитектура-услуг.md` (M1, M2),
`docs/architecture/23-seo-стратегия.md` §2–4, `docs/architecture/22-модель-данных.md` §5.

## Задача
Ежедневный автоматический пайплайн гороскопов + все программатические SSR-страницы +
шуточный контур. После этой фазы портал каждый день сам публикует свежий контент.

## Требования
1. **Пайплайн генерации** (worker, cron 00:10 МСК):
   - расчёт «астрособытий» дня/недели/месяца из astro-core (Луна: знак/ингрессы/void/лунный
     день; точные аспекты транзитных планет; ретро-статусы; фазы) → jsonb `astro_events`;
   - для каждого (знак × период × тема) — LLM-текст по шаблону промта: на входе астрособытия
     + правила тона (поддерживающий, конкретный, без фатализма и медицины, 600–900 знаков
     день / 1500–2500 неделя-месяц) + вариативность формулировок (антидубляж: запрет повторов
     зачинов из последних 7 выпусков — передавать список зачинов);
   - автомодерация (фильтры из Ф4) → status='published' | 'moderation' (очередь ручной);
   - идемпотентность: uniq-ключ; перегенерация только с флагом force.
2. **Экономия LLM**: тексты по знакам генерируются одним батч-промтом на период×тему
   (12 знаков за вызов) со структурированным выходом; кэш; месячные/годовые — реже.
3. **SSR-страницы** (по матрице док. 23 §2): `/goroskop/{znak}` (сегодня, по умолчанию),
   `/goroskop/{znak}/{period}`, тематические подстраницы; годовые `/goroskop/{yyyy}/{znak}`;
   восточный: хаб `/vostochnyj-goroskop/{yyyy}` + 12 страниц `/vostochnyj-goroskop/{yyyy}/{zhivotnoe}`
   (как в матрице 23 §2); лунные `/lunnyj-den/{n}`; хабы `/goroskop`,
   `/znaki-zodiaka/{znak}` (описание знака из вики-чанков + ссылки). Виджет «астропогода
   сегодня» на всех страницах. Перелинковка по правилам док. 23.
4. **Персональный гороскоп** (кабинет): транзиты к наталу пользователя → приоритизация
   (точность орбиса, значимость планет) → LLM-текст; краткий бесплатно, полный — подписка
   (пейвол-заглушка до Ф8); генерация лениво при заходе + кэш на день.
5. **Шуточный контур** (M2): `humor=true`; форматы «антигороскоп» и 2–3 профессиональных
   («гороскоп разработчика», «гороскоп бухгалтера»); отдельные страницы `/shutochnyj-goroskop/...`,
   явная маркировка «шуточный», отдельный тон-промт; OG-карточки для шеринга.
6. **page_cache**: SSR-рендер программатики кэшируется (тег `horoscope:{date_key}`),
   инвалидация после публикации; sitemap-индексы обновляются worker'ом; lastmod честный.
7. **Алерт**: если к 01:00 МСК дневной комплект не опубликован — алерт (протухшая
   программатика = удар по SEO).

## Верификация
- Прогон пайплайна за «завтра» в тест-режиме: 300 текстов сгенерированы, автомодерация
  прошла, страницы отдаются SSR со свежим контентом.
- Тексты дня привязаны к реальному небу: выборочно 5 текстов — упомянутые события есть в
  `astro_events` (антигаллюцинационная проверка Ф4 применима).
- Антидубляж: зачины текстов одного знака за 7 дней не повторяются (скрипт-проверка).
- Идемпотентность: повторный запуск cron не создаёт дублей и не дёргает LLM (кэш).
- Lighthouse программатической страницы ≥90 SEO; JSON-LD валиден (Rich Results Test).
- Ручная панель: 15 текстов (10 обычных + 5 шуточных), «плохо» <10%.
- Симуляция сбоя LLM в 00:10 → фоллбэк-провайдер или алерт сработал.
