# Промт Ф4 — LLM-конвейер, RAG, ИИ-разборы

## Контекст
Фаза Ф4 портала «Stassist». Прочитай: `docs/architecture/21-техническая-архитектура.md` §4,
`docs/architecture/22-модель-данных.md` §3, §6, `docs/research/07-ии-в-астрологии.md`
(анти-галлюцинационный консенсус — критично).

## Незыблемые правила
1. LLM НИКОГДА не вычисляет и не «вспоминает» позиции планет — все факты приходят из astro-core.
2. Каждый отчёт содержит блок «Расчётные данные» (что именно видел ИИ) и дисклеймер
   «информационно-развлекательный характер».
3. Тексты корпуса — только собственные/лицензионно чистые (синтез классики в общественном
   достоянии допустим; копирование современных авторов запрещено).

## Задача
`packages/llm` + корпус интерпретаций + ИИ-разборы и чат в кабинете.

## Требования
1. **Слой провайдеров**: интерфейс `LlmProvider` (generate, embed, стоимость, стриминг);
   адаптеры: Anthropic (приоритет), OpenRouter, YandexGPT, GigaChat; конфиг цепочки
   fallback через env; таймауты, ретраи с джиттером, circuit breaker; учёт токенов и
   стоимости в `ai_reports`.
2. **Сериализатор карты**: `ChartData` → компактный текстовый блок фактов (позиции со
   знаками/домами/градусами, аспекты с орбисами и аппликацией, достоинства, узлы/Лилит/Хирон,
   ретро, стеллиумы) — детерминированный, тестируемый снапшотами.
3. **Корпус интерпретаций** (`interpretation_chunks`): сгенерируй ядро корпуса —
   планета×знак (10×12), планета×дом (10×12), Asc×знак (12), мажорные аспекты×пары светил
   и личных планет (~120), арканы 1–22 × позиции матрицы (22×8 ключевых), накшатры (27,
   каркас) — процесс: LLM-черновик по жёсткому шаблону (тон: психологический, поддерживающий,
   без фатализма) → самопроверка вторым проходом → статус draft; редактор помечает reviewed.
   Эмбеддинги + HNSW-индекс. Версионирование чанков.
4. **Конвейер отчёта**: тип отчёта → необходимые расчёты (astro-core) → сериализация →
   точный ретрив чанков по ключам фактов (+ семантический для вопросов) → сборка промта
   (системные правила из §«Незыблемые правила» + данные + чанки + задача + структура ответа) →
   генерация → пост-валидация (zod-структура; фильтр запрещённого: медицина, смерть,
   гарантии событий, «порча» — regexp-слой + LLM-классификатор) → сохранение в `ai_reports`
   (кэш по `cache_key`).
5. **Типы разборов MVP**: `big3` (бесплатный), `natal_full` (по сферам: личность, отношения,
   карьера, ресурсы, задачи роста), `synastry`, `matrix_full`, `transit_month`.
6. **Чат** (`chat_sessions/messages`): вопрос → интент-роутер (какие расчёты нужны:
   натал/транзиты на дату/синастрия) → расчёт → конвейер; стриминг ответа (SSE);
   лимиты: free — 3 вопроса/день, премиум — fair use 100/день.
7. **PDF**: worker-задача: отчёт → HTML-шаблон (брендированный, с ChartWheel-SVG) →
   PDF (playwright-chromium или weasyprint-совместимый пайплайн) → S3, подписанные URL.
8. **Оценка качества**: golden-набор 20 карт → эталонные big3/natal_full; скрипт прогонки
   изменений промтов: LLM-judge сравнивает кандидата с эталоном по рубрике (соответствие
   данным карты! отсутствие выдуманных позиций, тон, структура) + отчёт для ручной панели.
   Кнопка «плохой разбор» в UI → `report_feedback`.

## Верификация
- Юнит: сериализатор (снапшоты), ретривер (точные ключи), фильтры запрещённого (набор
  провокационных кейсов: «когда я умру», «сниму ли порчу», медицинские вопросы → мягкий отказ
  с перенаправлением).
- Анти-галлюцинационный тест: 10 карт → в сгенерированных текстах НЕТ упоминаний позиций,
  отсутствующих в блоке данных (автопроверка регэкспами по названиям планет/знаков + ручная
  выборка).
- Failover-тест: отключи приоритетного провайдера — цепочка переключается, отчёт генерируется.
- Кэш: повторный запрос того же разбора не вызывает LLM (проверить по счётчику).
- E2E: профиль → big3 бесплатно → пейвол-заглушка на natal_full → (тест-режим) полный отчёт →
  PDF скачивается, содержит колесо, блок «Расчётные данные», дисклеймер.
- Панель качества: 10 отчётов вручную, «плохо» <10%; результаты в `_work/` отчёт фазы.
