# Промт Ф3 — Калькуляторы, SVG-карта, публичные инструменты

## Контекст
Фаза Ф3 портала «Stassist». Прочитай: `docs/architecture/20-архитектура-услуг.md` (M0, M6,
пользовательские пути), `docs/architecture/23-seo-стратегия.md` §2, §4,
`docs/architecture/22-модель-данных.md` §2 (calc_presets, charts) и §5 (astro_calendar, page_cache).

## Задача
Публичные интерактивные калькуляторы (лидогены) + изоморфный SVG-рендер карты.

## Требования
1. **`packages/ui/ChartWheel`** — SVG-колесо карты: знаки, дома, планеты (глифы), аспектные
   линии (цвет по типу), ретро-маркеры; тема светлая/тёмная; изоморфность (SSR + SPA + PNG
   через resvg в worker для OG-карточек). Конфигурация отображения из calc_preset.
2. **Публичные страницы-калькуляторы** (SSR + интерактивные острова):
   - `/natalnaya-karta` — форма (дата/время/место, «время неизвестно») → колесо + таблица
     позиций/аспектов + краткие текстовые блоки (из вики-чанков, пока заглушки) + CTA
     «сохранить карту и получить полный разбор» (→ регистрация);
   - `/sovmestimost` + `/sovmestimost/{znak-a}-i-{znak-b}` (78 канонических пар, зеркала — 301):
     статичный текст пары + виджет синастрии по датам рождения;
   - `/matrica-sudby` — октаграмма (SVG), арканы с краткими значениями, CTA на PDF;
   - `/kvadrat-pifagora`, `/chislo-puti` — расчёт + краткие значения;
   - `/lunnyj-kalendar` + `/lunnyj-kalendar/{yyyy-mm}` — сетка месяца из `astro_calendar`
     (лунные дни, фазы, знак Луны, void), «сегодня» — виджет.
3. **`astro_calendar`**: worker-задача предрасчёта на 18 месяцев вперёд (ежесуточное
   обновление скользящего окна) через astro-core.
4. **API**: анонимные расчётные эндпоинты `/api/v1/calc/*` (без сохранения ПД! расчёт на лету,
   ответ не логируется с входными данными), rate-limit, zod-схемы из shared.
5. **SEO-обвязка** (по док. 23): title/description-шаблоны, JSON-LD BreadcrumbList,
   canonical, sitemap для калькуляторов и пар совместимости; OG-карточка результата
   (генерация PNG по кнопке «поделиться», хранение в S3, без ПД на картинке — только
   позиции/колесо).
6. Гостевой сценарий: результат расчёта живёт в localStorage; после регистрации —
   предложение импортировать в профиль.

## Верификация
- E2E: каждый калькулятор от ввода до результата (включая «время неизвестно» — без домов);
  зеркальная пара совместимости отдаёт 301 на каноническую.
- Скриншот-тесты ChartWheel (3 эталонные карты, светлая/тёмная тема) — пиксельная стабильность.
- SSR-проверка: curl страницы пары совместимости содержит текст без JS; Lighthouse
  на `/natalnaya-karta` ≥ 90 perf/SEO (мобильный профиль).
- Проверить, что anon-расчёты не пишут дату рождения в логи/БД.
- PNG-шеринг: карточка генерируется ≤2 с, валидный OG-превью в Telegram (ручная проверка).
