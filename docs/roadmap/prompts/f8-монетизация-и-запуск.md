# Промт Ф8 — Монетизация, индивидуальные прогнозы, запуск

## Контекст
Финальная фаза MVP портала «Stassist». Прочитай: `docs/architecture/20-архитектура-услуг.md`
(M3, M9), `docs/architecture/22-модель-данных.md` §3–4, `docs/strategy/10-выводы-и-позиционирование.md`
§5 (тарифы), `docs/architecture/23-seo-стратегия.md`, `docs/research/05-рынок-услуги-юридика.md`.

## Задача
Реальный приём денег, подписка с пейволом, продукт «индивидуальный прогноз», перевод прода
в РФ и запуск.

## Требования
1. **Биллинг**: интеграция ЮKassa (первичный; CloudPayments — адаптер-заглушка по тому же
   интерфейсу): разовые платежи (PDF, прогнозы) и рекуррентная подписка (сохранённый метод);
   СБП; чеки 54-ФЗ через встроенную фискализацию провайдера; вебхуки — идемпотентны
   (`webhook_events.event_id` uniq), подпись проверяется; retry-очередь в worker;
   возвраты из админки; `payments`/`subscriptions` — по док. 22 §4.
2. **Подписка «Премиум»** (349 ₽/мес, 2 490 ₽/год, триал 7 дней): гейты — персональная
   полная лента (Ф5), безлимит чата и полные разборы (Ф4), скидка на PDF; статусы
   trial/active/past_due (грейс 3 дня)/cancelled; отмена в один клик из кабинета
   (юридическое требование прозрачности).
3. **Пейвол**: квиз-онбординг после первого расчёта (5–7 вопросов: сфера интереса, опыт,
   цели) → персонализированный экран пейвола (тексты вариируются по ответам);
   A/B-каркас: таблицы `experiments`/`experiment_events`/`quiz_answers` из док. 22 §7б;
   отписки рассылок — `email_optouts` (там же).
4. **Индивидуальные прогнозы (M3)**: мастер заказа (событие/период/вопрос → параметры →
   глубина) → оплата → конвейер Ф4 с расчётами по типу (транзиты на дату / карта периода /
   элективные окна в интервале) → PDF + онлайн-просмотр; статусная модель заказа полная
   (включая поля этапа 2 — expert_id остаётся null).
5. **Запуск-инфраструктура**: прод-БД — Yandex Managed PostgreSQL (миграции применяются
   с нуля; Supabase остаётся dev/стейджем с синтетикой); S3 — Yandex Object Storage;
   секреты прод-контура; бэкапы+восстановление отрепетированы; мониторинг/алерты (док. 21 §9);
   нагрузочный тест (autocannon: 200 RPS на программатику из кэша, 20 RPS на API).
6. **SEO-финализация**: снятие noindex, sitemap-индексы в Вебмастер/GSC, Метрика+цели
   (калькулятор→регистрация→оплата), robots, редиректы www/слэши, страница /methodology,
   контроль дублей. Сезонный контент: страницы «гороскоп 2027» к октябрю.
7. **Юридический чек-лист запуска**: оферта, политика ПД, согласие (отдельный документ),
   дисклеймер на отчётах/футере, реквизиты оператора ПД, уведомление РКН подано, баннер
   cookies; **проверка товарных знаков «Матрица судьбы» / «метод Ладини» в Роспатенте**
   (при конфликте — нейтральный нейминг PDF-продукта); мониторинг статуса законопроекта о
   запрете рекламы эзотерики (новая редакция от 13.02.2026); тексты — на вычитку юристу
   (блокер запуска).
8. **E-mail/push-цепочки**: брошенный расчёт (24 ч), конец триала (за 2 дня), еженедельный
   дайджест; отписка в один клик.

## Верификация
- Платёжный E2E в песочнице ЮKassa: разовый платёж → чек → PDF доступен; подписка → триал →
  списание → отмена → доступ закрыт по концу периода; возврат → доступ отозван.
- Вебхуки: повторная доставка события не задваивает платёж (идемпотентность-тест);
  подделка подписи → 401.
- Пейвол: оба A/B-варианта показываются, конверсия пишется в метрики.
- Индивидуальный прогноз: 3 типа заказов от оплаты до PDF; элективные окна в PDF
  соответствуют расчёту (сверка с astro-core напрямую).
- Прод-репетиция: полный деплой на чистую Yandex-инфраструктуру по runbook; восстановление
  БД из бэкапа ≤30 мин; нагрузочный тест выдержан (p99 SSR из кэша <300 мс).
- Финальный чек-лист: юридические страницы на месте; grep прод-логов — ПД/секретов нет;
  все ворота фаз Ф0–Ф7 зелёные на проде.
