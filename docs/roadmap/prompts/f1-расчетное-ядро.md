## astro-core: состав (реализуй в этом порядке)
1. **Время**: (дата, время, место) → UTC → юлианский день (JD UT + JD TT через ΔT-полиномы);
   исторические часовые пояса через IANA tzdb. Флаг `timeUnknown` → полдень локального времени,
   `noHouses: true`.
2. **Позиции**: astronomy-engine (MIT) — геоцентрические эклиптические долготы/широты/скорости
   Солнце–Плутон, Луна; ретроградность по знаку скорости. Для Солнца — модуль VSOP87
   (сгенерируй TS-таблицы скриптом `tools/gen-vsop87.ts` из свободных данных VSOP87D, точность ~1″).
3. **Точки**: средний и истинный лунные узлы, средняя Лилит (полиномы Меёса), Селена как
   точка напротив Лилит (конвенция московской школы; истинная/оскулирующая Лилит — версия 1.x,
   «Селена Глобы» с собственными эфемеридами — версия 2+, в MVP не делать); Хирон —
   скрипт `tools/gen-chiron.ts` тянет вектора из JPL Horizons API (1900–2100, шаг 1 сутки) →
   чебышёвские коэффициенты в собственный формат; рантайм интерполирует. Вне диапазона —
   явная ошибка `ChironOutOfRange`.
4. **Дома**: звёздное время (GMST/GAST) → RAMC → Asc/MC; системы: equal, whole sign, Porphyry —
   формулы; Regiomontanus, Koch — замкнутая тригонометрия; Placidus — итеративное деление
   полудуг (эталон Меёс). Широта >66.5° для Placidus/Koch → автофоллбэк Porphyry + флаг в meta.
5. **Аспекты**: мажорные+минорные, конфигурация орбисов по планетам/аспектам (пресеты школ);
   аппликация/сепарация по скоростям.
6. **Root-finding** (Брент по разности долгот): точные транзитные аспекты в интервале, ингрессы,
   void-of-course Луны (последний птолемеев аспект в знаке → ингресс), фазы, соляры/лунары
   (возврат к натальной долготе; Солнце — по VSOP87). Лунные дни — по восходам Луны для
   геолокации (SearchRiseSet).
6б. **Прогностика без новой астрономии** (обязательно в Ф1 — используется модулями M0/M3/M4
   и заявлена в charts.kind): вторичные прогрессии («день за год»: JD рождения + возраст×1 сутки),
   символические дирекции (+k°/год), композит (средние точки пар по короткой дуге + композитный MC),
   карта Davison (середина по времени и месту).
7. **Сидерика/джйотиш** (расчётные функции «про запас» — продуктовый UI джйотиша выйдет в 1.x,
   но ядро делаем сразу, чтобы не перевскрывать): айанамши (Лахири дефолт, Раман, КП,
   Фаган—Брэдли, Юктешвар); накшатры; вимшоттари-даша (периоды/подпериоды). Ба-цзы: отдельный
   модуль — 4 столпа по солнечному календарю (стволы/ветви, 5 элементов).
8. **API**: `computeChart(input: ChartInput): ChartData` + функции прогностики; zod-схемы
   `ChartInput`/`ChartData` в `packages/shared` (единый источник); `ChartData.meta` — версия
   ядра, система домов, айанамша, флаги фоллбэков, оценка точности.
9. **Арабские части**: Фортуна (день/ночь по секте) и каркас для добавления других.

## numerology-core
Число жизненного пути (сквозная редукция, мастер-числа 11/22 сохраняются), числа
выражения/души/личности (таблица кириллицы — циклическая 1–9; правила Ё/Й задокументированы
в README и константах), психоматрица Пифагора (4 доп. числа, сетка 3×3, строки/столбцы/диагонали),
персональный год/месяц/день, совместимость по числам (lookup 9×9), **Матрица судьбы**:
октаграмма (вершины: день, месяц, свёртка года ≤22, сумма; центр; прямой квадрат; линии
отношений/денег; карта здоровья по 7 чакрам; 4 предназначения) — сверь с 2–3 публичными
калькуляторами на 10 датах.

## Верификация (ворота фазы)
- **Golden-тесты**: ≥30 эталонных карт (эпохи 1900–2050, широты вкл. >60°, южное полушарие,
  timeUnknown; фикстуры собери вручную сверкой с astro.com и sotis-online.ru, сохрани в
  `packages/astro-core/test/golden/` с указанием источника и даты съёма). Допуски: планеты ≤1′,
  Asc/MC ≤2′, куспиды ≤2′, фазы/ингрессы ≤2 мин, момент соляра ≤1 мин. Расхождения сверх
  допуска — либо чинить, либо документировать в `ACCURACY.md` с обоснованием.
- Property-тесты: сумма размеров домов = 360°; долготы непрерывны; ретро ⇔ скорость < 0;
  синастрия симметрична.
- Матрица судьбы: 10 дат сверены с matricaladini.ru (или аналогом) — расхождений нет.
- Производительность: полный расчёт натала ≤50 мс, соляр ≤200 мс (бенчмарк в CI).
- Лицензионный аудит: `license-checker` в CI — только разрешённые лицензии.
