# 22. Модель данных

> Пакет документов портала «Зодиакум» · 12.07.2026
> PostgreSQL 15+, Drizzle ORM, SQL-first миграции. Нотация ниже — псевдо-Drizzle (сокращённо).
> Все таблицы: `id uuid pk default gen_random_uuid()`, `created_at/updated_at timestamptz`.
> ПД-поля помечены 🔒 (шифрование на уровне приложения) — см. [21, §7](21-техническая-архитектура.md).

## 1. Пользователи и доступ

```ts
users:            email (uniq, citext), email_verified_at, password_hash /*argon2id*/,
                  display_name, avatar_key /*S3*/, role enum('user','editor','moderator','admin','expert'),
                  status enum('active','blocked','deleted'), tz, locale default 'ru'
refresh_tokens:   user_id fk, token_hash (uniq), family_id uuid /*reuse detection: семья сессий*/,
                  expires_at, rotated_from fk nullable, revoked_at, ip_hash, ua_hint
password_resets:  user_id fk, token_hash, expires_at, used_at
consents:         user_id fk, kind enum('pd_processing','marketing'), doc_version, granted_at,
                  revoked_at   /*152-ФЗ: отдельное согласие, версия документа*/
audit_log:        actor_id fk nullable, action, entity, entity_id, payload jsonb, request_id
```

Reuse detection: при погашении refresh создаётся потомок (`rotated_from`); предъявление уже
ротированного токена ⇒ отзыв всех токенов `family_id`.

## 6. Вики и корпус интерпретаций (RAG)

```ts
wiki_articles:    slug uniq, section enum('planets','signs','houses','aspects','schools',
                  'nakshatras','arcana','lunar_days','stones','glossary'),
                  title, body_md, status, editor_id fk, version int, seo jsonb
interpretation_chunks:
                  key uniq /*'planet_in_sign:mars:leo' | 'aspect:sun:square:moon' |
                  'planet_in_house:venus:7' | 'arcanum:3:destiny' | 'nakshatra:7' …*/,
                  tradition enum('western','vedic','karmic','numerology') default 'western',
                  text, source_article_id fk nullable, embedding vector(1024),
                  quality enum('draft','reviewed'), version int
stones:           slug uniq, name, properties_md, colors text[], zodiac_signs text[],
                  planets text[], decades jsonb, arcana int[], chakras text[],
                  suitable_md, unsuitable_md, photo_keys text[],
                  affiliate_url nullable /*задел партнёрской витрины, этап 2*/
```

Ретрив в ИИ-конвейере: точный (по `key` из фактов карты) + семантический (embedding вопроса).
Чанки версионируются — отчёт хранит версии использованных фрагментов (аудит качества).

без даты/места) — публикация не раскрывает данные рождения.

## 8. ER-сводка (основные связи)

```
users 1─n birth_profiles 1─n charts n─1 calc_presets
users 1─n orders n─1 ai_reports; orders n─1 users(expert)   ← задел маркетплейса
users 1─n subscriptions n─1 plans; payments n─1 orders|subscriptions
wiki_articles 1─n interpretation_chunks → (pgvector) → ai_reports/chat
horoscopes, astro_calendar, compat_pages → page_cache (SSR-программатика)
posts n─1 charts(анонимные); friendships → синастрия по разрешению
```

## 9. Индексация и объёмы (ориентиры MVP)

- `horoscopes(scope,sign,period,topic,date_key)` uniq — основной паттерн выборки программатики.
- `interpretation_chunks.embedding` — HNSW; ожидаемый объём корпуса: 10–40 тыс. чанков.
- `charts.data` jsonb — GIN не нужен (доступ по id); объём ~5–20 КБ/карта.
- `payments.idempotency_key`, `webhook_events.event_id` — уникальные (идемпотентность).
- Партиционирование `audit_log`, `chat_messages` по месяцам — при росте, не в MVP.
