> дополнительно проверены адверсариальной верификацией — статусы см. [00-обзор-исследования.md](00-обзор-исследования.md).

## 1. Ядро расчёта натальной карты

Конвейер: дата/время + часовой пояс → UTC → **юлианский день** (JD в UT + перевод в TT через ΔT) → **позиции планет** (видимая геоцентрическая эклиптическая долгота на эпоху даты: нутация, аберрация, световое время) → **звёздное время** (GMST/GAST + долгота → LST → RAMC) → **Asc/MC** → **куспиды домов** → аспекты.

- **Системы домов.** Равнодомная и целых знаков — тривиальны (Asc/знак Asc + шаг 30°). Регимонтан и Кох — замкнутые тригонометрические формулы. **Плацидус** — итеративный алгоритм (деление полудуг по времени, нет решения в замкнутой форме; эталон — Meeus, «Astronomical Algorithms»). Плацидус и Кох **ломаются за полярным кругом (~66.5°)** — нужен fallback (Swiss Ephemeris в таких случаях подставляет Порфирия — требует проверки конкретного поведения). `swe_houses_ex()` в Swiss Ephemeris покрывает ~25 систем одним вызовом.
- **Аспекты/орбисы** — чистая геометрия (разность долгот против набора углов 0/60/90/120/180 + минорные, орбисы конфигурируются по планетам/аспектам). **Ретроградность** — знак скорости по долготе (Swiss Ephemeris возвращает скорость напрямую; иначе — численная производная).
- **Узлы** (средний — полином Меёса; истинный — сложнее), **Лилит** (средний/оскулирующий апогей Луны; `swe_nod_aps()` в SE даёт всё), **Хирон** — только по эфемеридным файлам `seas_*.se1` в Swiss Ephemeris; в Moshier-режиме и в astronomy-engine Хирона **нет**. **Арабские части** — арифметика: Фортуна = Asc + Луна − Солнце (день) / Asc + Солнце − Луна (ночь).

## 2. Прогностика, совместимость, хорар, лунный календарь

- **Транзиты**: позиции на момент T против натала; точные аспекты — поиск корня по разности углов (в SE 2.10 есть `swe_solcross`/`swe_mooncross`). **Вторичные прогрессии**: «день за год» — JD_рождения + возраст_в_годах × 1 сутки. **Символические дирекции**: +k° в год ко всем точкам (чистая арифметика). **Соляр/лунар**: итерация (Ньютон, 2–3 шага) момента, когда транзитное Солнце/Луна возвращается к натальной долготе.
- **Синастрия** — кросс-аспекты двух карт (обычно суженные орбисы). **Композит** — средние точки пар планет по короткой дуге + композитный MC; **Davison** — карта на середину времени/места.
- **Хорар** — обычная карта на момент вопроса; специфика (луна без курса, via combusta, достоинства планет, «радикальность» Asc) — таблицы и логика, астрономии не требует.
- **Лунный календарь**: фазы — элонгация Луна−Солнце (0/90/180/270); **лунные дни** (русская традиция): границы по восходам Луны, 1-й день — от новолуния до первого восхода → нужны восходы для геолокации; **луна без курса (void)** — момент последнего точного птолемеевского аспекта Луны в знаке до ингресса в следующий знак (поиск корней по каждой паре Луна—планета + момент ингресса).

## 5. astronomy-engine: точность и что дописывать

Точность **±1 угловая минута** относительно NOVAS/JPL Horizons на тысячи лет; тела: Солнце—Плутон, Луна, галилеевы спутники, кастомные малые тела по векторам состояния. Есть: фазы Луны, SearchMoonQuarter, восходы/заходы (SearchRiseSet), Ecliptic-преобразования, затмения, апсиды Луны, SearchMoonNode/NextMoonNode (моменты прохождения узлов). **Дописывать придётся**: Asc/MC и все системы домов (~300–500 строк по Меёсу), средний/истинный узел и среднюю Лилит как долготы (полиномы Меёса), аспектный движок, поиск ингрессов/точных аспектов (root-finding поверх позиций), арабские части. **Хирон недоступен в принципе** — реалистичный обход: предрассчитать позиции из JPL Horizons (данные свободны) в собственную таблицу/чебышёвские коэффициенты на 1900–2100.

**Какая точность реально нужна**: орбисы аспектов — градусы, поэтому для трактовок хватает угловых минут с запасом. Куспиды домов: 1° ошибки = 4 минуты времени рождения — погрешность времени рождения почти всегда доминирует над эфемеридной. Тайминг событий: Луна идёт ~33″/мин → ошибка 1′ даёт ~2 мин в моменте фазы/void (для календаря ок); для соляра ошибка 1′ по Солнцу даёт ~24 мин по времени карты → дома соляра поплывут, здесь запас astronomy-engine минимален. Главный практический риск — **несовпадение с astro.com в последних минутах дуги**, на что жалуются пользователи; Swiss Ephemeris — отраслевой эталон соответствия. VSOP87 (данные свободно распространяются) — ~1″ и лучше на ±2000 лет, порты есть (vsop87-multilang), но это только планеты без всей обвязки.

## Ключевые факты
- Swiss Ephemeris лицензируется двойственно: до версии 2.10.0 включительно — GPL-2.0, с версии 2.10.1 (2021) — AGPL-3.0, что для любого закрытого SaaS означает обязанность открыть весь код либо купить профессиональную лицензию.
- Официальный прайс astro.com (июль 2026): Swiss Ephemeris Professional Edition — 700 CHF разовым платежом; встречающиеся в сторонних источниках цифры 750/400/1550 CHF и срок 99 лет — требуют проверки по контракту с Astrodienst.
- Текущая версия Swiss Ephemeris — 2.10.3 (апрель 2026, данные пересобраны на JPL DE441); точность 0.001 угловой секунды, диапазон 13201 г. до н.э. — 17191 г. н.э.
- npm-пакет sweph (timotejroiko) — актуальные и активно поддерживаемые биндинги SE для Node.js (v2.10.3-7, N-API, prebuilt-бинарники, TypeScript-типы, 100% покрытие API); при наличии профлицензии используется под LGPL-3.0.
- npm-пакет swisseph (mivion) фактически заброшен: версии 0.5.x, систематические сбои сборки node-gyp на Node 20+ и Python 3.12 — для нового проекта непригоден.
- swisseph-wasm (prolaxu) — живая WASM-обёртка (браузер и Node, GPL-3.0-or-later), но автор не аффилирован с Astrodienst, и для закрытого коммерческого использования профлицензию всё равно нужно покупать у Astrodienst.
- astronomy-engine — MIT-лицензия, точность ±1 угловая минута против NOVAS/JPL Horizons, ~117 КБ без зависимостей; но нет систем домов, аспектов, Лилит, арабских частей и Хирона — их придётся писать самим.
- Хирон вычислим только по эфемеридным файлам seas_*.se1 Swiss Ephemeris; в Moshier-режиме SE и в astronomy-engine его нет (обход — предрасчёт из бесплатного JPL Horizons в собственную таблицу).
- Встроенный Moshier-режим Swiss Ephemeris (без файлов на диске) даёт 0.1″ по планетам и 3″ по Луне; JS-порты Moshier (npm ephemeris v2.2.0, Moshier-Ephemeris-JS) распространяются под GPL-3.0.
- VSOP87 обеспечивает ~1″ и лучше на ±2000 лет от J2000 (полные ряды — ~0.1″), данные распространяются свободно; усечённые ряды Меёса дают около 1″.
- Для трактовок хватает точности в угловые минуты (орбисы аспектов — градусы); ошибка времени рождения доминирует: 4 минуты времени = ~1° сдвига Asc и куспидов.
- Для тайминга: ошибка 1′ по Луне даёт ~2 минуты погрешности момента фазы/аспекта (приемлемо для календаря), но 1′ по Солнцу даёт ~24 минуты погрешности времени соляра — дома солярной карты заметно плывут.
- Плацидус считается только итеративно (эталон — алгоритм Меёса) и, как и Кох, не работает выше ~66.5° широты — нужен fallback (например, на Порфирия); равнодомная и целых знаков тривиальны.
- Луна без курса рассчитывается как интервал от последнего точного птолемеевского аспекта Луны (0/60/90/120/180) в текущем знаке до ингресса в следующий знак — реализуется поиском корней по разности долгот.
- Лунные дни в русской традиции отсчитываются по восходам Луны (1-й день — от новолуния до первого восхода), то есть требуют расчёта восходов Луны для геолокации пользователя (в astronomy-engine есть SearchRiseSet).
- Соляры/лунары считаются итерацией Ньютона по моменту возврата Солнца/Луны к натальной долготе; в SE 2.10 для этого есть готовые функции swe_solcross/swe_mooncross.
- Вторичные прогрессии («день за год»), символические дирекции (+k° в год), композит (средние точки) и синастрия (кросс-аспекты) не требуют новой астрономии — только обвязка над расчётом позиций.
- circular-natal-horoscope-js — готовая натальная библиотека под Unlicense (public domain), но по точности и поддержке уступает связке Swiss Ephemeris/astronomy-engine.
- Использование старых GPL-2.0 версий SE (≤2.10.0) на сервере без раскрытия кода — юридически серая зона (ASP-loophole), противоречащая заявленной воле Astrodienst; для публичного коммерческого продукта не рекомендуется.
- Итоговая рекомендация исследователя — купить профлицензию SE (~700 CHF) + sweph; **решением заказчика вариант отклонён: принят open-source путь** (astronomy-engine + собственные дома, см. 21 §3).
