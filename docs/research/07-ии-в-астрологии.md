## 1. Существующие AI-астро-продукты

**Nebula (OBRIO, Украина)** — крупнейший западный игрок: 30+ млн пользователей, топ-1 lifestyle-приложение в App Store/Google Play в США/Канаде/Австралии. Оценки трекеров: ~90 тыс. загрузок и ~$300 тыс. выручки/мес на iOS, ~100 тыс. и ~$200 тыс. на Android (оценки, требует проверки). Модель — гибридная: расчёты по эфемеридам NASA, базовый AI-чат-бот, но основная монетизация — платные консультации с 250+ живыми консультантами; компания публично заявляла, что не заменит советников машинами.

**Co-Star (США, осн. 2017, Нью-Йорк, $15 млн инвестиций)** — данные NASA JPL для позиций планет; интерпретации — заранее написанные людьми (астрологи + поэты) фрагменты, собираемые алгоритмически (NLP-стек: spaCy, BERT/CyBERT). Не полноценный LLM-чат. В 2022 получил волну критики за «пугающие» push-уведомления и генерализации (fear-marketing).

**Kundli GPT / Vedic AstroGPT / Melooha (Индия, ведическая астрология)** — Kundli GPT: бесплатный чат-бот, заявляет расчёт по Swiss Ephemeris + интерпретация по классическим принципам (раши, накшатры, даши, йоги). Vedic AstroGPT — freemium, «проприетарный LLM» (маркетинговое заявление, требует проверки). Melooha (Бангалор, VC-финансирование) — «200+ проприетарных алгоритмов» + реальные астрономические данные + дообучение на фидбэке пользователей. JyothishGPT — заявляет fine-tuning GPT-3.5 через LlamaIndex.

**Русскоязычный рынок**: Telegram-боты — Natalon (Swiss Ephemeris, премиум от 100 ₽, AI-разборы по темам), Астарио, AstroLuck, MatrikaSoulBot; веб — AiBro, AstroWorld AI, AstroGPT; для профи — астропроцессор Chronos, Sotis Online. Показателен кейс AIPanda (Хабр): Swiss Ephemeris + дома Плацидуса, расчёт 3–5 сек, GPT-4 Turbo, RAG по базе из «тысяч фрагментов астрологических текстов», ~5000 регистраций/мес.

**Западные LLM-native сервисы**: My Zodiac AI (Swiss Ephemeris + LLM, 10 языков), NatalChart.AI, Natal AI, OrbitWise («пайплайн из 7 агентов»), Nummi, ERA.

## 2. Типовая архитектура: расчёт вне LLM

Консенсус индустрии — **строгое разделение слоёв**:

1. **Эфемеридный расчёт** — Swiss Ephemeris или NASA JPL: позиции планет, узлов, Хирона, куспиды домов, аспекты. Детерминированно, никогда не через LLM.
2. **Структурированные данные карты** — фиксированные схемы (JSON/XML), валидация координат до передачи дальше.
3. **Сериализация для LLM** — данные карты преобразуются в компактное текстовое/XML-описание («ground truth»). Эталонные open-source-примеры: библиотека **Kerykeion** (Python, 140 тыс. загрузок/мес, модуль `context_serializer` → XML для LLM) и **chart2txt**.
4. **RAG по базе интерпретаций** — извлечение релевантных фрагментов (планета в знаке/доме, аспекты) и вставка в промт с инструкцией «отвечать строго по предоставленным данным». Продвинутый вариант — **GraphRAG** (планеты/дома/аспекты как граф сущностей).
5. **LLM-генерация текста** — GPT-4o/Claude/Gemini синтезируют связную интерпретацию.

Готовые API берут на себя слои 1–4: AstrologyAPI (3000+ продуктов, 50 стран — по их маркетингу), VedicAstroAPI (интерпретации «написаны астрологами вручную», 21 язык вкл. русский, режим BYO-LLM), Prokerala, Astrologer API (на Kerykeion, есть «AI Context»-эндпоинты), VedIntel (Swiss Ephemeris + Claude). Оценка стоимости разработки (OmiSoft): MVP $7–10 тыс., средний уровень $15–25 тыс., продвинутый $30 тыс.+ (оценки вендора).

## 3. Галлюцинации LLM и борьба с ними

«Голый» LLM не умеет считать карту: числа для него — токены, а не координаты. Луна проходит ~13°/сутки, асцендент сдвигается ~1° за 4 минуты — вероятность угадать близка к нулю. Типичная картина: солнечный знак верный, Луна/асцендент/дома выдуманы. Тест AllAboutAI (Gemini 1.5 Pro, GPT-4o, Claude 3.5 Sonnet, Perplexity; рубрика на 100 баллов): лучший результат — Gemini 1.5 Pro, 81/100, но и он ошибался в ключевых позициях планет; Claude силён в интерпретации, слаб в расчёте.

Методы защиты: (а) расчёт полностью вне LLM (эфемеридный движок); (б) валидация выходов по схемам перед промтом; (в) RAG вместо свободной генерации; (г) явные инструкции в промте («NEVER invent a planetary position» — так делает open-source-проект aryaminus/astro: детерминированный движок ~2930 строк Python, 18 MCP-инструментов, 31 REST-эндпоинт, 3 традиции — западная/ведическая/БаЦзы); (д) редакторская вычитка профессиональными астрологами.

## 4. База знаний интерпретаций и авторские права

**Открытые/свободные источники**: общественное достояние — классика: Уильям Лилли «Christian Astrology» (1647), Алан Лео (ум. 1917) — доступны на Project Gutenberg/Internet Archive. Открытые корпуса под свободными лицензиями редки: aryaminus/astro содержит MIT-лицензированные «оригинальные синтезы классических источников» (8 справочных документов — рулсеты, а не копии текстов); TkMidpoint — interpretations.xml по мидпоинтам. Крупного открытого датасета «планета в знаке/доме + аспекты» в свободном доступе НЕ обнаружено — это дефицит.

**Авторские тексты**: корпуса известных астрологов защищены. Прецеденты лицензирования: отчёты Лиз Грин на astro.com — копирайт совместно у Liz Greene, Astrodienst AG и Astro*Intelligence AG (модель «автор + платформа», начата в 1985–87); «Planets in Transit» Роберта Хэнда (1976, 720 делинеаций) десятилетиями лицензируется в report-writer-софт (Astrolabe). Astrodienst не продаёт API и не лицензирует тексты Грин третьим лицам. Практика рынка: либо заказ собственного корпуса у астрологов (как VedicAstroAPI), либо лицензирование у автора, либо синтез из классики. Обучение LLM на пиратских PDF (Сакоян-Эккер и др. широко «гуляют» в сети) — юридический риск.

**Важно для лицензий кода**: Swiss Ephemeris — двойная лицензия: AGPL v3 (копилефт распространяется на SaaS/веб-API — придётся открыть весь код) или коммерческая лицензия Astrodienst: по актуальному прайсу «Edition June 2026» — CHF 700 разово, unlimited (старые цифры 750/400 устарели, см. поправки в 02). Kerykeion — тоже AGPL-3.0.

## 5. Оценка качества AI-интерпретаций

Строгих академических бенчмарков именно для астро-LLM не найдено (arXiv-работы касаются астрономии). Практика индустрии:
- **Рубрики с баллами** (AllAboutAI: 100-балльная шкала — техническая точность карты + качество интерпретации);
- **Многокритериальные сравнения** (My Zodiac AI: движок расчёта, глубина синтеза vs шаблонность, языки, диалоговость, цена, trial);
- **Редакторская проверка астрологами** до/после запуска;
- **Пользовательский фидбэк** и дообучение (Melooha);
- Заявленная «точность 70–80% на узких тестах» (AllAboutAI) — методологически слабая метрика, требует проверки.

**Выводы для проекта**: (1) расчёт — только детерминированным движком, LLM никогда не считает; (2) карта → строгая JSON/XML-схема → RAG-промт; (3) собственный лицензионно чистый корпус интерпретаций — ключевой актив и главный дефицит рынка; (4) расчётное ядро — только open-source (зафиксированное решение заказчика: astronomy-engine + собственные дома, см. 21 §3); Swiss Ephemeris без AGPL/платной лицензии недоступен; (5) контроль качества — рубрики + панель астрологов + фидбэк-луп.

## Ключевые факты
- Консенсусная архитектура AI-астрологии: данные рождения → эфемеридный расчёт (Swiss Ephemeris/NASA JPL) → структурированные данные карты (JSON/XML со схемой) → RAG-промт с базой интерпретаций → LLM-текст; позиции планет никогда не вычисляются самим LLM.
- Голый LLM систематически галлюцинирует карту: солнечный знак обычно верен, но Луна (~13°/сутки) и асцендент (~1° за 4 минуты) выдумываются; в тесте AllAboutAI лучший из четырёх LLM (Gemini 1.5 Pro) набрал лишь 81/100 и ошибался в позициях планет.
- Nebula (OBRIO, Украина) — 30+ млн пользователей, гибридная модель: эфемеридные расчёты + базовый AI-чат, но монетизация через 250+ живых консультантов; оценки выручки ~$300 тыс./мес iOS и ~$200 тыс./мес Android (данные трекеров, требуют проверки).
- Co-Star (осн. 2017, $15 млн инвестиций) использует данные NASA JPL и заранее написанные людьми фрагменты интерпретаций, собираемые алгоритмически (spaCy/BERT), без полноценного LLM-чата; в 2022 критиковался за пугающие push-уведомления.
- Kundli GPT, Vedic AstroGPT, Melooha (Индия) строят ведическую AI-астрологию: Swiss Ephemeris + классические принципы (даши, накшатры, йоги); Melooha заявляет 200+ проприетарных алгоритмов и дообучение на фидбэке (маркетинговые заявления).
- Русскоязычный рынок активен: Telegram-боты Natalon (Swiss Ephemeris, премиум от 100 ₽), Астарио, AstroLuck, MatrikaSoulBot; веб-сервисы AiBro, AstroWorld AI; кейс AIPanda с Хабра: Swiss Ephemeris + Плацидус + GPT-4 Turbo + RAG по тысячам фрагментов астротекстов.
- Против галлюцинаций применяют: расчёт вне LLM, валидацию по схемам, RAG (включая GraphRAG по графу планеты-дома-аспекты), жёсткие промт-инструкции типа 'NEVER invent a planetary position' и редакторскую вычитку астрологами.
- Kerykeion — ключевая open-source библиотека (Python, ~140 тыс. загрузок/мес, AGPL-3.0) с модулем context_serializer, преобразующим данные карты в XML 'ground truth' для LLM; на ней построен коммерческий Astrologer API.
- Open-source проект aryaminus/astro: детерминированный движок ~2930 строк Python без зависимостей, 18 MCP-инструментов, 31 REST-эндпоинт, три традиции (западная, ведическая, БаЦзы) и MIT-лицензированные синтезы классических источников — редкий пример свободного корпуса интерпретаций.
- Swiss Ephemeris распространяется по двойной лицензии: AGPL v3 (копилефт распространяется на SaaS — нужно открыть весь код сервиса) или коммерческая лицензия Astrodienst за CHF 750 (первая) + CHF 400 (последующие).
- Крупного открытого датасета интерпретаций 'планета в знаке/доме + аспекты' не существует — в свободном доступе только классика в общественном достоянии (Уильям Лилли 1647, Алан Лео) и небольшие MIT/XML-корпуса; собственный корпус — дефицитный актив.
- Тексты известных астрологов защищены копирайтом: отчёты Лиз Грин принадлежат совместно Liz Greene, Astrodienst AG и Astro*Intelligence AG; 'Planets in Transit' Роберта Хэнда (720 делинеаций) лицензируется в report-writer-софт с 1970-80-х — модель 'автор + платформа' является отраслевой практикой.
- Astrodienst (astro.com) не предоставляет коммерческого API и не лицензирует свои интерпретационные тексты третьим лицам — только библиотеку Swiss Ephemeris.
- Готовые астро-API с интерпретациями: AstrologyAPI (заявляет 3000+ продуктов в 50 странах), VedicAstroAPI (тексты 'написаны астрологами вручную', 21 язык включая русский, режим BYO-LLM), Prokerala, VedIntel (Swiss Ephemeris + Claude).
- Качество AI-интерпретаций оценивают нестрого: балльные рубрики (100-балльная шкала AllAboutAI), многокритериальные сравнения (точность движка, глубина синтеза vs шаблоны, языки, диалоговость), панели астрологов и пользовательский фидбэк; академических бенчмарков для астро-LLM не найдено.
- Оценка стоимости разработки AI-астро-приложения (вендор OmiSoft): MVP $7-10 тыс., средний уровень с RAG $15-25 тыс., продвинутый уровень $30 тыс.+ (требует проверки).
- Появляются мультиагентные архитектуры: OrbitWise использует 'пайплайн из 7 AI-агентов', каждый отвечает за свой аспект карты.
- Типичное время полного расчёта натальной карты через Swiss Ephemeris в продакшене — 3-5 секунд (кейс AIPanda).
